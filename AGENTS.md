# morning-bot — AGENTS.md

## 0. Назначение и приоритет правил

Этот файл — глобальный контракт проекта для Claude/Codex.

При конфликте используй порядок:

1. Явное текущее задание владельца
2. `AGENTS.md`
3. Актуальный `docs/<feature>.md`
4. Реальный код и стиль соседних модулей
5. `RELEASE_NOTES.md` — только история, не спецификация

Если `AGENTS.md` и feature-doc расходятся, исправь документацию вместе с кодом.

Главный принцип:

> Бот не должен ощущаться как набор функций. Он должен ощущаться как личный помощник, который говорит коротко, по делу и вовремя.

## 2. Проект

`morning-bot` — персональный Telegram-бот на Python.

Стек:

- Python
- `python-telegram-bot`
- PostgreSQL KV
- in-memory fallback без `DATABASE_URL`
- Railway
- `MessageBuilder`
- централизованный AI-слой

Базовые команды:

```bash
python -m pip install -r requirements.txt
python bot.py
python -m py_compile path/to/changed_file.py
python scripts/sync_version.py
```

Railway запускает:

```bash
python bot.py
```

Не хардкодить секреты, токены, лимиты и настройки, которые уже есть в env,
`config.py`, `settings.py` или runtime-слое.

# UI SYSTEM

## 4. Навигация

Во всём боте:

```text
⬅️ Назад
#️⃣ Главная
```

Не использовать:

```text
#️⃣ Меню
⬅️ Назад в ...
⬅️ Админ
⬅️ Система
```

`⬅️ Назад` возвращает на предыдущий логический экран.

`#️⃣ Главная` всегда ведёт в главное меню.

Нижняя строка по умолчанию:

```text
⬅️ Назад | #️⃣ Главная
```

`#️⃣ Главная` не должна удалять полезный результат из истории.

## 6. Единый словарь кнопок

### Добавление

```text
🆕 Добавить ...
```

### Удаление / очистка

```text
❌ ...
```

Не использовать `🗑`.

### Персональные параметры

```text
🎚️ Предпочтения
```

### Функциональные / системные параметры

```text
🎚️ Настройки
```

Старый паттерн `*️⃣ Выбрать ...` не использовать как универсальную кнопку.

### Новая рекомендация

Если результата ещё нет:

```text
✨ Подобрать ...
```

Если результат уже показан:

```text
✨ Другой ...
```

Для поездок:

```text
✨ Куда поехать
✨ Другая поездка
```

Не использовать `✨ Заменить`, если действие можно назвать по смыслу.

### Любимое и сохранение

```text
❤️ Любимое = уже нравится
💾 Сохранить = на потом
✅ Сохранено = уже сохранено
```

Не объединять эти состояния.

# AI & DATA

## 13. AI policy

LLM не используется по умолчанию.

Перед вызовом:

1. Можно решить правилами?
2. Можно собрать шаблоном?
3. Есть данные в store?
4. Есть свежий кэш?
5. Нужны персонализация, свободный текст или объяснение?

LLM подходит для:

- персональных рекомендаций
- «почему тебе»
- разбора свободного текста
- языковых объяснений
- суммаризации
- сложного fallback

LLM не нужен для:

- меню
- кнопок
- пагинации
- удаления
- сортировки
- статусов
- подсчётов
- простого форматирования
- админских лимитов

---

## 14. Общий AI-слой

Feature-модули не должны самостоятельно выбирать модель, если есть общий слой.

Общий слой отвечает за:

- provider routing
- fallback
- cooldown
- кэш
- лимиты
- logging
- rate limits
- нормализацию ошибок

Конкретный порядок провайдеров хранить централизованно, а не дублировать здесь.

По возможности передавать:

- `feature`
- `user_id`
- `input_hash`
- `cache_ttl`
- короткий prompt
- fallback без AI

---

## 15. Structured output

LLM по возможности возвращает данные, не готовый Telegram-текст.

Telegram UI собирает код.

Повторно используемые prompts держать централизованно в `prompts/`.

---

## 16. Кэш

Не делать внешний запрос при свежем кэше.

Кэш должен учитывать только реально влияющие параметры:

```text
user_id
feature
language
level
city
input_hash
profile_version
date_bucket
```

TTL и бюджеты держать централизованно.

При изменении входных данных пользователя кэш должен инвалидироваться через
новый hash/version или явную очистку.

---

## 17. Пользовательские данные

- не терять данные при UI-рефакторинге
- не перезаписывать весь профиль ради одного поля
- новые поля имеют fallback
- старые профили продолжают работать
- миграции осторожные
- одинаковые сущности не дублировать без причины

## 27. 🧠 Тренажёр

Тренажёр — непрерывная адаптивная очередь.

Нет:

- фиксированных 12 заданий
- автоматического завершения
- итогового экрана
- обязательной длительности

Пользователь сам выходит через `⬅️ Назад` или `#️⃣ Главная`.

После результата:

```text
✨ Другое задание
```

Не использовать `🔄 Следующее задание`.

### Форматы

Оставить:

1. `🎯 Выбери перевод`
2. `🧠 Вспомни`
3. `✏️ Вставь слово`
4. `🧩 Собери фразу`
5. `🔍 Найди ошибку`
6. `🗣 Скажи в ситуации`
7. `💭 Что ответить`

Убрать как отдельные форматы:

- `💬 Продолжи диалог`
- `💬 Естественный вариант`
- `🔤 Формы глагола`

Глагольные формы можно хранить в словаре.

### 🎯 Выбери перевод

Telegram Quiz:

```text
изучаемый язык → русский
```

### 🧠 Вспомни

Telegram Quiz:

```text
русский → изучаемый язык
```

Это выбор, не свободный ввод.

Активное воспроизведение проверяет `🗣 Скажи в ситуации`.

### ✏️ Вставь слово

Для слова, предлога, артикля или части конструкции.

Если слово отсутствует — это не `Найди ошибку`.

### 🔍 Найди ошибку

Только когда неправильное слово / форма реально присутствует.

### 🧩 Собери фразу

Только когда порядок слов полезен.
Не использовать для двухсловных сочетаний.

### 🗣 Скажи в ситуации

Свободный ввод.
Принимать естественные корректные альтернативы.

### 💭 Что ответить

Для разговорных реплик.
Только один ответ должен естественно подходить.

### Ошибки и повтор

После ошибки:

1. сохранить результат
2. показать короткий правильный ответ
3. вернуть материал позже
4. дать минимум 3 других задания
5. повторить обычно через 3–5 заданий другим подходящим форматом

Не давать тот же материал сразу после показа полного ответа.

### Результаты

```text
✅ Верно
🟡 Почти
❌ Ошибка
📝 Ответ
```

`🟡 Почти` — только небольшая ошибка в свободном ответе.

После Quiz не показывать огромную словарную карточку.

SRS сохраняется независимо от отсутствия фиксированной сессии.

UI-заголовок:

```text
🕵️ Детектив
```

Загадка — на активном языке.

После решения — короткий разбор причин, без длинного `Analyse:`.

Если используется фото:

- один главный объект
- без стаи / толпы
- без коллажей
- объект крупный
- если хорошего фото нет, лучше без фото

---

## 29. 📖 Словарь

Правила:

- термин и перевод в одной строке через `→`
- не дублировать `Перевод:`
- один частотный короткий пример
- не добавлять дубли
- TTS не меняет карточку при ошибке
- повторный TTS использует кэш, если возможно

---

## 30. 🚑 Здоровье

Главный экран — безопасный короткий фокус дня.

Подразделы:

```text
👩🏻‍⚕️ Врач
💊 Лекарства
😮‍💨 Мысли
🎚️ Предпочтения
```

Главный экран не ставит диагноз и не назначает лечение.

В `Мысли` использовать:

```text
🧐 Разобрать мысли
🕒 Оставить на потом
❌ Очистить записи
```

Не писать `Удалить мысли`.

---

## 31. ✈️ Поездки

Если рекомендация уже показана:

```text
✨ Другая поездка
🧳 Мои поездки
🎚️ Предпочтения
#️⃣ Главная
```

Карточка страны должна быть конкретной.

Не писать шаблоны:

- «подходит пользователю»
- «уникальное сочетание»
- «отношение терпимое»

Использовать полезные блоки:

```text
✨ Тебе подойдёт
📍 Не пропусти
☀️ Когда ехать
💶 Бюджет
👩🏻‍🏫 Языки
🏳️‍🌈 LGBTQ+
```

Актуальные факты и безопасность должны быть проверяемыми.

## 32. 🍿 Досуг

Главный экран при наличии данных показывает одну персональную рекомендацию дня,
а не объяснение функций.

Кнопки:

```text
🎫 Концерты
🎬 Кино
🎧 Музыка
📖 Книги
#️⃣ Главная
```

Эти эмодзи не менять.

## 33. 🎬 Кино

Главный экран содержит компактный блок:

```text
🎟️ Сейчас в кино · Нидерланды
```

Показывать максимум 5 фильмов.

Остальное открывать через:

```text
🎟️ Весь прокат
```

Кнопки:

```text
🎟️ Весь прокат

🌙 По настроению
❤️ Моё кино
💾 Смотреть позже
```

```text
✨ Другое кино
❤️ В любимые / ❤️ В любимых
💾 Смотреть позже / ✅ Сохранено
```

## 34. 📖 Книги

```text
🎚️ Предпочтения

#️⃣ Главная
```

Не блокировать подбор при пустом списке любимых.

Карточка:

- одно короткое описание
- максимум два пункта «Почему стоит читать»
- короткий сюжет без спойлеров
- `✨ Другая книга`

## 35. 🎧 Музыка

Сохранять именно:

```text
🎧 Музыка
```

Кнопки:

```text
✨ Подобрать музыку
❤️ Мои артисты
💾 Послушать позже
🎚️ Предпочтения
⬅️ Назад
#️⃣ Главная
```

Карточка артиста:

- конкретное звучание
- максимум два пункта «Почему тебе»
- 2–3 точки «С чего начать»
- один короткий проверяемый факт
- `✨ Другой артист`
- `🎫 Концерты`

Язык обучения может быть дополнительным сигналом, но не должен ломать совпадение
с музыкальным вкусом.

## 36. 🎫 Концерты

Сохранять именно:

```text
🎫 Концерты
```

Главный экран сразу показывает ближайшие события.
Отдельный экран `🔥 Ближайшие` не нужен.

Карточка события компактная:

```text
Romy
21 августа · Biddinghuizen 🇳🇱
Lowlands 2026 · Фестиваль
✅ Подтверждён · Подробнее…
```

Не повторять подписи `Формат`, `Место`, `Дата`.

Кнопки:

```text
❤️ Любимые артисты
🔍 Найти артиста
🌍 Нидерланды
⬅️ Назад
#️⃣ Главная
```

На кнопке страны показывать текущую страну.

`Подробнее…` должно реально открываться.

## 39. Админка

Админка компактная.

### Система

Формат строки:

```text
🟢 Service · статистика
🟡 Service · причина → резерв
🔴 Service · недоступно
```

Если сервис глобальный, не писать `Везде`.

У зелёного не писать `работает`, `доступен`, `активен`, `в норме`.

Статус отражает возможность следующего запроса.

Raw errors и traceback — только во вложенных логах.

### Пользователи

Список короткий:

```text
🟢 Света · 1 день назад
🟢 Vladimir · 2 дня назад
🔴 Дарь · не заходил
```

Удаление — только внутри карточки пользователя с подтверждением.

## 40. Рабочий алгоритм

Перед:

1. Найди реальный модуль
2. Найди UI
3. Прочитай feature-doc
4. Посмотри соседний код
5. Определи минимальный scope

Во время:

1. Меняй только нужное
2. Не ломай callback-и
3. Не меняй соседний UX без причины
4. Не вызывай AI напрямую при наличии общего слоя
5. Не теряй данные
6. Соблюдай единый UI vocabulary

После:

1. Выполни дешёвую проверку
2. Обнови feature-doc при изменении поведения
3. Обнови `RELEASE_NOTES.md` для законченной значимой правки
4. Дай короткую ручную проверку
5. Честно скажи, что проверено

## 42. Definition of Done

- [ ] UX не стал сложнее без причины
- [ ] Везде правильный `⬅️ Назад`
- [ ] Везде `#️⃣ Главная`, а не `#️⃣ Меню`
- [ ] Нет старого универсального `*️⃣ Выбрать ...`
- [ ] Добавление использует `🆕`
- [ ] Удаление / очистка используют `❌`
- [ ] Кнопки отражают текущее состояние
- [ ] Нет технического мусора в пользовательском UI
- [ ] Callback-и не конфликтуют
- [ ] UI не читает store и не вызывает AI
- [ ] AI идёт через общий слой
- [ ] Свежий кэш не обходится лишним сетевым запросом
- [ ] Пользовательские данные сохранены
- [ ] Feature-doc актуален
- [ ] Release notes обновлены, если правка законченная и значимая
- [ ] Выполнена подходящая техническая проверка
- [ ] Результат проверки описан честно

Если применимый пункт не выполнен — задача не готова.
