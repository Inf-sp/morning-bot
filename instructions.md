# DM — Telegram-бот личного ассистента

Персональный бот-компаньон на каждый день. Совмещает сводку дня, гардероб, изучение языков, досуг, здоровье и свободный чат в одном месте.

---

## Команды

| Команда | Описание |
|---|---|
| `/start` | Приветствие + главное меню (нижняя reply-клавиатура) |
| `/setup` | Настройки (город, язык, уведомления, уровни, параметры тела, списки) |
| `/notes` | Закладки и любимое |

Команды регистрируются в Telegram через `post_init → set_my_commands` ([bot.py:502-508](bot.py#L502-L508)).

## Архитектура

### Точка входа и обработчики

Ядро — [bot.py](bot.py): на `python-telegram-bot` в режиме long-polling регистрируются обработчики и три ежедневных job'а, затем вызывается `app.run_polling(drop_pending_updates=True)` ([bot.py:511-527](bot.py#L511-L527)). Вся доменная логика вынесена в тематические модули, а `bot.py` работает диспетчером.

Зарегистрировано 6 обработчиков ([bot.py:513-519](bot.py#L513-L519)):

| Обработчик | Назначение |
|---|---|
| `CommandHandler` ×3 | `/start`, `/notes`, `/setup` |
| `CallbackQueryHandler` | все инлайн-кнопки (`answer_callback`) |
| `MessageHandler(filters.LOCATION)` | геолокация → `weather.location_handler` |
| `MessageHandler(filters.Document.ALL)` | `.txt`-файл одежды → `document_handler` |
| `MessageHandler(filters.TEXT)` | текстовый роутер + fallback в свободный чат |

### Маршрутизация инлайн-кнопок

`answer_callback` ([bot.py:47-311](bot.py#L47-L311)) разбирает `callback_data` по префиксам и делегирует в модуль. Схема префиксов:

| Префикс | Куда уходит |
|---|---|
| `as_food*`, `as_daycheck`, `as_motiv`, `as_doctor` | `balance.handle_callback` |
| прочие `as_*` | `notes.handle_callback` (закладки/любимое) |
| `w_*` | `wardrobe.handle_callback` |
| `md_*` | `myday.handle_callback` |
| `set_*` | `handle_settings` (внутри bot.py) |
| `m_*` | навигация по подменю (`menu.menu_screen`, редактирует то же сообщение) |
| `a_*` | одноразовые действия (большой `if/elif` ~40 веток) |
| `lvl_`, `gram_`, `again_`, `next_gram_`, `game*`, `gamelang_`, `gamediff_` | обучение |
| `movie_`, `book_`, `listen_`, `reco_` | досуг |
| `worry_*`, `chat_retry` | тревоги / повтор |

### Текстовый роутер и конечный автомат

`text_router` ([bot.py:314-395](bot.py#L314-L395)) обрабатывает текст со **строгим порядком приоритетов** — это ключевая деталь:

1. нажатие кнопки нижнего reply-меню сбрасывает `pending_input` (чтобы чат не «съел» сообщение настроек);
2. кнопки «☀️ Мой день» / «👕 Гардероб» и ярлыки `LABEL_TO_KEY` открывают раздел;
3. режим добавления одежды файлом (`add_wardrobe_mode`);
4. **игра и обратный перевод проверяются ДО `pending_input`** — иначе ответ ушёл бы не туда;
5. `pending_input` — словарь «жду конкретный ввод» (город, тревоги, слово, параметры тела, страна/артист/книга, и т.д.); ключ снимается через `pop` и обрабатывается соответствующей веткой;
6. всё остальное → свободный чат `assistant.chat_reply`.

По сути бот — **роутер + конечный автомат состояний**: кнопка ставит флаг `store.pending_input[cid] = "..."`, следующее текстовое сообщение этот флаг снимает и направляется в нужный обработчик.

### Автоматические уведомления (job_queue)

Три `run_daily` по таймзоне `Europe/Amsterdam` ([bot.py:522-524](bot.py#L522-L524)); каждый шлёт в `CHAT_ID` только если тип включён в настройках (`settings.notif_on`):

| Время | Job | Что делает |
|---|---|---|
| 11:00 | `job_grammar` | `learning.send_morning_word` — порция слов по методу дня недели |
| 14:00 | `job_checkin_day` | ставит `pending_input="worry"` и просит выгрузить тревоги текстом |
| 20:00 | `job_checkin_evening` | `balance.send_evening_review` — разбор тревог на «факт»/«отпустить» |

Включаются/выключаются в `/setup → Уведомления`.

## Модули

### `ai.py` — LLM-слой

Отказоустойчивая цепочка из 6 провайдеров. `llm()` перебирает их по порядку и возвращает первый непустой ответ ([ai.py:90-108](ai.py#L90-L108)):

- `DEFAULT_ORDER = claude → openai → gemini → openrouter → groq → cf`
- `LEARN_ORDER = openai → claude → …` (для обучения)

Модели: Claude `claude-sonnet-4-6`, Gemini `gemini-2.5-flash` (с `thinkingBudget: 0`), Groq / Cloudflare — Llama, OpenAI / OpenRouter — из env. При полном провале `_friendly()` возвращает человекочитаемое сообщение и отдельно ловит rate-limit (429) ([ai.py:80-85](ai.py#L80-L85)).

Три обёртки:
- **`llm(prompt, max_tokens, temperature, order)`** — простая генерация.
- **`llm_json()`** ([ai.py:154-175](ai.py#L154-L175)) — структурированный ответ. Просит «только валидный JSON без markdown», затем чинит результат в 5 нарастающих по агрессивности попыток, включая самописный `_repair_inner_quotes()`, который посимвольно экранирует незакрытые кавычки внутри строк ([ai.py:110-152](ai.py#L110-L152)). Защита от «почти-JSON» от модели.
- **`chat_chain(history)`** ([ai.py:252-261](ai.py#L252-L261)) — свободный чат с системным промптом, та же цепочка фолбэков.

`CHAT_SYSTEM` ([ai.py:177-204](ai.py#L177-L204)) зашивает персону пользователя и жёсткие правила формата: только Telegram-HTML (запрет Markdown), «суть вперёд» (адаптация под СДВГ), максимум 1 эмодзи, без встречных вопросов и «рассказать подробнее». В конец подмешивается `config.LAGOM`.

### `store.py` — состояние и хранилище

Два уровня хранения:

- **Персистентное** — key-value поверх PostgreSQL (`DATABASE_URL`), таблица `kv (key TEXT PRIMARY KEY, value JSONB)` ([store.py:9-57](store.py#L9-L57)). Если БД недоступна — прозрачный откат в словарь `_mem` в памяти. Несмотря на `.json`-имена ключей (`settings.json`, `dict.json` и т.д.), это **не файлы**, а ключи в БД/памяти.
- **Эфемерное** — глобальные словари в памяти, сбрасываются при рестарте ([store.py:126-143](store.py#L126-L143)): `chat_history`, `game_state`, `game_config`, `grammar_state`, `challenge_state`, `pending_input`, `add_wardrobe_mode`, `last_recos`, `last_answer`, `last_source`, `last_recipe`, `last_action`, `recent_looks`, `del_index`, `last_word`, `game_recent`, `suggested_countries`. Держат «контекст последнего шага» (например, `last_answer` нужен кнопке «⭐ В закладки», `recent_looks` — чтобы не повторять образ 3 дня).

Хелперы: `get_settings/set_settings` (город), `get_level/set_level` (уровень языка), `load_wardrobe/save_wardrobe/merge_wardrobe` (шкаф с версионированием по полю `_v` из сида `wardrobe.json`), `get_list/add_to_list/set_list` (произвольные списки по ключу).

### `config.py` — конфигурация и сиды

Читает env-переменные, задаёт `TZ = Europe/Amsterdam`, дефолтный город (Алкмар, NL) и имена всех storage-ключей ([config.py:24-47](config.py#L24-L47)). Подгружает сид-файлы и шаблоны промптов:
- `countries.json` → `VISITED` (посещённые страны);
- `prompts.json` → `STYLE_PROFILE`, `TEMP_ZONES`, `CONTENT_TASTE`;
- `lagom.json` → `LAGOM` (установки тона/ценностей для системного промпта).

`myday_rules()` / `place_hint()` ([config.py:77-110](config.py#L77-L110)) собирают правила для генерации сводки дня, при этом для NL дают расширенную подсказку про регион (NS, велоправила, налоги).

### `menu.py` — меню

Нижняя reply-клавиатура `MAIN_KB` (5 разделов) ([menu.py:6-10](menu.py#L6-L10)) и инлайн-подменю через `menu_screen(key)` ([menu.py:25-77](menu.py#L25-L77)). «Мой день» и «Гардероб» открываются сразу; «Баланс», «Обучение», «Досуг» — через `LABEL_TO_KEY` показывают инлайн-экран. Подменю вложенные: `m_balance → m_food`, `m_learn → m_nl/m_en`.

### `util.py` — утилиты форматирования

- `esc()` — экранирование `& < >` для HTML.
- `tg_html()` ([util.py:35-74](util.py#L35-L74)) — **санитайзер**: чистит ответ модели под Telegram-HTML (markdown `**`/`*`/`__` → теги, заголовки `#` убираются, маркеры списков → «• »), сохраняет только разрешённые теги. Страховка поверх системного промпта.
- `send_html()` — отправка с откатом на plain text, если HTML невалиден.
- `flag_from_cc()` — эмодзи флага из ISO-кода страны; `country_flag()` — флаг по названию через LLM.

### `assistant.py` — свободный чат

`chat_reply` ([assistant.py:10-29](assistant.py#L10-L29)): хранит последние 10 сообщений на чат (`chat_history`, обрезка `hist[-10:]`), показывает «печатает…», прогоняет через `chat_chain`, сохраняет ответ в `last_answer` (для закладок). Триггерный детектор: если в тексте есть медицинские слова (`боль`, `симптом`, `врач`…), снизу предлагает кнопку «👩🏻‍⚕️ Вопрос врачу».

### `myday.py` — ☀️ Мой день

`send_plany` ([myday.py:147-158](myday.py#L147-L158)) собирает сводку дня с **кэшем на сутки** (`_day_cache` per chat, ключ — дата). Сводка (`_build_day_text`) объединяет:
- погоду на сегодня (из `weather.fetch_weather`, иконка, ветер, периоды дождя);
- слово дня (случайное из личного словаря `DICT_KEY`);
- интересный факт о локации + цитату — генерируются разом через `plany_extras` → `ai.llm_json` по правилам `config.myday_rules` (факт обязан быть реальным и про текущий город);
- образ (`outfit`) генерируется, но в сводке не показывается — нужен для будущих фич.

Кнопки под сводкой: погода на завтра / на неделю, сменить город, «⭐ В закладки» (`md_*` — сохранение факта или цитаты в `NOTES_KEY`). `ensure_lagom()` подгружает Лагом-практики из `lagom.json`. Смена города сбрасывает кэш через `reset_day_cache`.

### `weather.py` — погода

Источник — Open-Meteo (`fetch_weather`, осадки/ветер/температура почасово и посуточно) ([weather.py:22-31](weather.py#L22-L31)). Геокодинг города — Open-Meteo Geocoder с фолбэком на Nominatim, по нескольким вариантам написания и языкам (`set_city_text`). Обратный геокодинг локации — BigDataCloud (`location_handler`).

Логика отображения:
- `weather_icon` / `wind_scale` / `_rain_real` — иконки и шкала ветра; дождь показывается только при вероятности ≥ 50% и осадках ≥ 0.1 мм;
- `_periods` — раскладывает дождь/ветер по «утром/днём/вечером/ночью»;
- `send_weather(mode)` — режимы `today` / `tomorrow` / `week` / `full`;
- `storm_alert` ([weather.py:165-192](weather.py#L165-L192)) — штормовое предупреждение при ветре > 15 м/с, снеге или ливне; для NL добавляет «Code Geel», NS и Buienradar;
- `_world_fact` — берёт 4 случайные мировые точки, тянет реальную текущую температуру и через LLM делает шутливый факт про самую экстремальную;
- `_meteo_fact` — исторический метео-факт про текущую страну (для NL — «архивы KNMI»), со строгим запретом на прогнозы и выдумки.

### `wardrobe.py` — 👕 Гардероб

- **`send_looks`** ([wardrobe.py:57-94](wardrobe.py#L57-L94)) — образ по погоде через `ai.llm_json`, строго из вещей шкафа, с жёсткими температурными правилами и `config.STYLE_PROFILE`. Не повторяет образы за 3 дня (`recent_looks`). Поддерживает сценарии «Официальная» / «Вечеринка» (`SCENARIOS`).
- **`send_show`** — шкаф, сгруппированный по зонам (Верх/Низ/Обувь/Аксессуары) через словарь ключевых слов `ZONES`.
- **`add_item`** — парсинг текста/файла в категории через LLM и `merge_wardrobe`.
- **`send_del` / `del_item`** — удаление вещи по индексу (`del_index`).
- **`send_improve`** — критический разбор гардероба (что убрать / заменить / фактура / аксессуары).
- **`check_purchase`** — «модный приговор» БРАТЬ / НЕ БРАТЬ по стилю и шкафу.

Результаты пишут `last_answer`/`last_source` для закладок.

### `balance.py` — 🧠 Баланс

- **Кулинарный радар**: `send_recipe` (завтрак/обед/ужин) с карточкой и кнопкой «Полный рецепт» (`last_recipe`), `send_leftovers` — блюдо из остатков (`pending_input="leftovers"`). Учитывает технику пользователя (электроплита, духовка SAGE).
- **Личная мотивация** (`_gen_motiv`) — микро-техники для СДВГ, один шаг прямо сейчас; одноразовая генерация с «🔄 Ещё» (`_ONESHOT`).
- **Вопрос врачу** (`doctor_answer`, [balance.py:160-191](balance.py#L160-L191)): если вопрос про лекарство (`_is_med_question`) — справка по препарату строгого формата; иначе генерирует 6 тезисов-кандидатов, прогоняет через `rerank.rerank` (ZeroEntropy, top-3) и отдаёт структурированный разбор симптомов. Везде дисклеймер «не диагноз».
- **Дневник тревоги**: `send_daycheck` (сбор), `send_evening_review` ([balance.py:230-267](balance.py#L230-L267)) — LLM раскладывает каждую тревогу на «Факт / Предположение» + итог дня; `worry_mark` / `worry_clear_all` / `show_worry_check` — отметка «подтвердилось/не сбылось» с прогресс-баром ментальной разгрузки. При закрытии всех тревог итог пишется в `DIARY_KEY`.

`retry()` ([balance.py:360-389](balance.py#L360-L389)) восстанавливает «Продолжить/Ещё» по `last_action` (oneshot / recipe / leftovers / role / чат).

### `learning.py` — 📚 Обучение

- **Грамматика** (`send_grammar`, [learning.py:60-139](learning.py#L60-L139)): задание с выбором A/B, объяснение, пример в настоящем и прошедшем, правило. Тема — из изучаемых тем пользователя или новая по уровню. Для NL ≥ B1 ориентируется на учебник TaalCompleet. «Ещё пример» (та же тема) и «Следующая тема» (новая).
- **Обратный перевод** (`do_translate` / `translate_answer`): RU-фраза → перевод на язык, проверка через `llm_json` с разбором ошибки и естественным вариантом. Состояние в `challenge_state`.
- **Пословица** (`send_proverb`) — оригинал + дословный перевод + русские аналоги + значение.
- **Словарь** ([learning.py:229-340](learning.py#L229-L340)): раздельно NL/EN, слова и фразы. Добавление нормализует через LLM (для NL — с артиклем de/het). Подгрузка дефолтов из `dict_nl.json`. Редактирование/удаление по индексу.
- **Daily Words** (`send_morning_word`, 11:00): метод по дню недели (`WEEK_TRACK` — загрузка / повтор / разгрузка / проверка / отдых на выходных) + порция до 5 слов из словаря.
- **Изучаемые темы** (`send_topics` / `add_topic`): пользователь добавляет тему, бот сразу даёт мини-разбор. Хранятся раздельно `TOPICS_NL_KEY` / `TOPICS_EN_KEY`.
- **Игра-детектив** (`send_game`, [learning.py:449-582](learning.py#L449-L582)): угадай персонажа по подсказкам. Настраиваемый язык (RU/EN/NL) и сложность (easy/med/hard), локализованный UI (`GAME_UI`), подсказка, «сдаюсь», нечёткое сравнение ответа (`_fuzzy`), 2 попытки, не повторяет последних 30 (`game_recent`).
- **Уровни** (`send_levels`) — A1…C2 раздельно для каждого языка, влияют на сложность.

### `content.py` — 🍿 Досуг (фильмы, книги, музыка, концерты)

- **Фильмы/сериалы** (`send_recos`/`movie`): LLM рекомендует 5 под профиль вкуса (`config.CONTENT_TASTE` + референсы из watchlist), затем `_tmdb_lookup` обогащает карточку (постер, жанры, рейтинг, описание), `_pick_good_movie` фильтрует по рейтингу ≥ 7.0 и исключает уже виденное/чёрный список (`MOVIE_BLACKLIST_KEY`). Кнопки: не нравится / в любимые / в закладки. Мусорные совпадения TMDb (`making of`, тизеры) отсекаются.
- **Книги** (`send_books_reco`): референсы из «Мои книги» (`BOOKS_KEY`, сид из `content.json`), обложки из OpenLibrary. Карточка с сюжетом, цитатой, «почему зайдёт».
- **Музыка** (`send_listen`): один новый артист под вкус (`artists.json`), исключая любимых/закладки/дизлайки (`MUSIC_DISLIKE_KEY`), до 3 попыток получить действительно нового. Карточка: звучание, на кого похоже, с чего начать, факт.
- **Концерты** (`find_concerts`, [content.py:533-642](content.py#L533-L642)): Ticketmaster по артистам пользователя, в домашней стране или выбранной из 12 европейских (`CC_MAP`). Фильтрует трибьюты/cover/candlelight (`TRIBUTE`), дедуплицирует по (артист, дата), показывает площадку/город/дату/ссылку.

### `travel.py` — ✈️ Путешествия

`send_go` ([travel.py:46-72](travel.py#L46-L72)) предлагает 1 новую страну, исключая посещённые (`COUNTRIES_KEY` / сид `countries.json`), закладки и дизлайки (`TRAVEL_DISLIKE_KEY`), до 3 попыток. `send_plan` — подробный план (почему подойдёт, лучшее время, бюджет из домашнего города, что не пропустить, LGBTQ+-дружелюбность, факт). `save_plan` сохраняет полный план в закладки (`NOTES_KEY`, `full: True`). `travel_fav` — в «Мои страны».

### `notes.py` — 💾 Закладки и любимое (`/notes`)

Два «ведра» в одной ленте `NOTES_KEY`, различаются полем `bucket`:
- **⭐ Временные закладки** (`bucket=fav`) — любой ответ бота сохраняется кнопкой «⭐ В закладки» (`save_fav` сжимает текст через LLM `_shorten`). При удалении типизированной закладки (фильм/книга/музыка/страна) предлагается «🚫 В чёрный список» / «❤️ В любимые» / «🗑 Удалить» — маршрутизация по `_note_type`.
- **❤️ Любимые** (`send_love_home`) — постоянные категорийные списки: страны, артисты, книги, одежда (шкаф), Лагом-практики. С добавлением/удалением и авто-загрузкой дефолтов.
- **📤 Экспорт** (`export_notes`) — выгрузка всех сохранений в `.txt`-файл.

### `settings.py` — ⚙️ Настройки (`/setup`)

Собственный namespace `user_settings.json` (поверх `store._load/_save`). Разделы:
- **Уведомления** — тумблеры по трём типам (`notif_<kind>`), 🟢/⚪;
- **Язык грамматики** — нидерландский / английский (`study_lang`);
- **Уровни** — делегирует в `learning.send_levels`;
- **Город** — текст (`pending_input="setcity"`) или геолокация;
- **Параметры тела / стиль** (`send_body`) — рост/вес/размеры + стиль (минимализм / скандинавская эстетика / натуральные ткани), используются при подборе образа;
- **Списки** — «Мои страны», «Мои артисты», «Мои книги» (добавление/удаление, авто-сид);
- **Шкаф** — общий с разделом «Гардероб».

### `rerank.py` — ранжирование

`rerank(query, documents, top_n)` ([rerank.py:6-27](rerank.py#L6-L27)) — обёртка над ZeroEntropy (`zerank-2`). Используется только в «Вопросе врачу» для отбора 3 самых релевантных тезисов. Режет документы до 600 символов и до 800 штук. Без ключа — бросает исключение (вызывающий код имеет фолбэк).

## Разделы меню (пользовательский вид)

### ☀️ Мой день
Сводка дня (погода, слово дня, факт о городе, цитата), дневник тревоги, переходы к погоде на завтра/неделю, смена города.

### 👕 Гардероб
Образ по погоде (+ сценарии), просмотр шкафа по зонам, добавление вещей текстом/файлом, разбор гардероба, оценка покупки.

### 🧠 Баланс
Вопрос врачу, личная мотивация (СДВГ), дневник тревоги, кулинарный радар (завтрак/обед/ужин + остатки).

### 📚 Обучение
NL/EN: грамматика (A/B), обратный перевод, изучаемые темы, пословицы; словарь (слова/фразы); игра-детектив; уровни A1–C2.

### 🍿 Досуг
Фильмы/сериалы (TMDB), книги (OpenLibrary), музыка, концерты (Ticketmaster), путешествия.

### ⚙️ Настройки (`/setup`)
Уведомления, язык грамматики, уровни, город, параметры тела/стиль, списки (страны/артисты/книги), шкаф.

## Хранилище и сид-файлы

Данные лежат в PostgreSQL (`kv`-таблица) или в памяти (откат). При первом обращении пустые списки засеваются из файлов репозитория:

| Сид-файл | Засевает |
|---|---|
| `wardrobe.json` | стартовый гардероб (версия в поле `_v`) |
| `dict_nl.json` | словарь нидерландских слов |
| `artists.json` | любимые исполнители |
| `content.json` | «Мои книги» (секция `books`) |
| `countries.json` | посещённые страны |
| `lagom.json` | Лагом-практики и установки тона |
| `prompts.json` | шаблоны промптов (стиль, темпер. зоны, вкус) |

## Технический стек

- **Python** — `python-telegram-bot` (long-polling), `job_queue` для расписания
- **AI** — цепочка фолбэков: Claude (`claude-sonnet-4-6`), OpenAI, Gemini (`gemini-2.5-flash`), OpenRouter, Groq, Cloudflare Workers AI
- **Ранжирование** — ZeroEntropy (`zerank-2`, rerank) — только «Вопрос врачу»
- **Погода** — Open-Meteo (прогноз + геокодинг), Nominatim (фолбэк геокодинга), BigDataCloud (обратный геокодинг)
- **Контент** — TMDB (фильмы/сериалы), OpenLibrary (обложки книг), Ticketmaster (концерты)
- **Хранилище** — PostgreSQL (`kv` JSONB) с откатом в память; сиды — JSON-файлы репозитория
- **Часовой пояс** — `Europe/Amsterdam`

## Переменные окружения

| Переменная | Назначение | Обязательна |
|---|---|---|
| `TELEGRAM_TOKEN` | Токен бота | да |
| `GEMINI_API_KEY` | LLM (Gemini) | да |
| `ANTHROPIC_API_KEY` | Claude | нет |
| `OPENAI_API_KEY` / `OPENAI_MODEL` | OpenAI (по умолч. `gpt-4o-mini`) | нет |
| `OPENROUTER_API_KEY` / `OPENROUTER_MODEL` | OpenRouter (по умолч. `meta-llama/llama-3.3-70b-instruct:free`) | нет |
| `GROQ_API_KEY` | Groq | нет |
| `CLOUDFLARE_API_TOKEN` / `CLOUDFLARE_ACCOUNT_ID` | Cloudflare Workers AI | нет |
| `CHAT_ID` | Telegram chat ID для авто-уведомлений | для рассылок |
| `DATABASE_URL` | PostgreSQL (иначе хранение в памяти) | нет |
| `ZEROENTROPY_API_KEY` | rerank в «Вопросе врачу» | нет |
| `TICKETMASTER_API_KEY` | Концерты | нет |
| `TMDB_API_KEY` | Фильмы/сериалы (постеры, рейтинг) | нет |

Без опциональных ключей соответствующие фичи деградируют мягко: фолбэк на следующего LLM-провайдера, текст вместо постера, подсказка о недостающем ключе для концертов.
