Deploy-уведомления стали предсказуемее: версия релиза теперь повышена для нового деплоя, а описание обновления вынесено в отдельный свежий блок.
Новости для тебя: раздел сразу показывает список свежих новостей, поиск расширен по всем категориям, а лишние кнопки периода, тем и ручного обновления убраны.

## v1.8.3

Deploy-уведомления стали удобнее: версия теперь хранится в проекте и обновляется вместе с кодом, поэтому Railway больше не нужно настраивать вручную для каждого релиза.
Новости для тебя: админская проверка свежих источников больше не упирается в cooldown и запускается сразу.
Админ-панель: убраны отдельные вкладки «Диагностика» и «Тесты», проверка перенесена в «Систему», а тестовые уведомления встроены в «Уведомления» с короткими кнопками по времени отправки.
Документация: добавлено глобальное правило, что любая кнопка возврата во всём боте называется только «⬅️ Назад».

## v1.8.2

Новости для тебя: убраны служебные сообщения, добавлен поиск по стране и fallback на найденные источники, чтобы раздел не показывал пустую карточку.

Админ-панель: переработана навигация и UX вокруг экранов «Система», «Диагностика», «Уведомления», «Пользователи», «Тесты» и «Логи». Главный экран стал короткой сводкой без debug-шума, подробности API/LLM/новостей перенесены в диагностику, тестовые отправки стали короткими inline-действиями только для администратора.
## v1.16.212 · Вложенные экраны

## 2026-07-23 · Навигация музыки и концертов

- Концерты открываются только внутри Музыки, а экран Досуга стал компактным списком из трёх категорий
- «Мои артисты» и «Послушать позже» объединены в одну строку, лишние действия на экране концертов убраны
## 2026-07-23 · Рассылка без автоматического разбора мыслей

- В вечернем уведомлении убрана кнопка «🧐 Разобрать мысли»; разбор по-прежнему доступен внутри раздела «Мысли»
## 2026-07-23 · Мысли открываются сразу с разбором

- При входе в «Мысли» активные записи сразу получают бережный общий разбор с конкретным следующим шагом
- Кнопка «❌ Очистить записи» остаётся на экране результата
## 2026-07-24 · Снижение фоновой нагрузки

- KV-мутации PostgreSQL повторно используют одно соединение процесса вместо нового подключения на каждую запись
- Фоновые проверки внешних провайдеров выполняются реже, а процессный TTL-кэш ограничен по размеру
- В Railway build context исключены тесты, кэши, документация и служебные каталоги
- Ежедневный прогрев главных экранов ограничен пользователями с включённой утренней сводкой; технические логи индикаторов ожидания переведены в DEBUG
