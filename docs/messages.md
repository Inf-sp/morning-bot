# Сообщения

Эта инструкция — для тех, кто хочет самостоятельно поправить текст, эмодзи или оформление
сообщений, которые бот присылает в Telegram, не разбираясь глубоко в коде.

## 1. Где искать нужный текст

Все тексты сообщений бота собраны в одной папке — **`ui/`**. Внутри неё — по одному файлу
на каждый раздел бота:

| Файл в `ui/`         | За что отвечает |
|-----------------------|------------------|
| `myday.py`             | Сводка «Мой день» (погода, слово дня, факт, совет, цитата) |
| `weather.py`           | Прогнозы погоды на сегодня/завтра/неделю, штормовые предупреждения |
| `wardrobe.py`          | Гардероб, подбор одежды |
| `food.py`              | Рецепты, холодильник |
| `learning.py`          | Тренажёр языка, игра в слова, поговорка дня |
| `dictionary.py`        | Личный словарь |
| `leisure.py`           | Фильмы, книги, музыка, путешествия |
| `balance.py`           | Дневник тревог, вечерний обзор дня, здоровье |
| `settings.py`          | Настройки, избранное, список Сохранить, админ-раздел |
| `onboarding.py`        | Знакомство с ботом при первом запуске |
| `menu.py`              | Главное меню |
| `assistant.py`         | Свободные ответы ассистента |
| `builder.py`           | Не тексты, а «инструменты», из которых собираются все сообщения (см. ниже) |

**Правило:** если хотите поправить текст какого-то сообщения — сначала определите, к какому
разделу бота оно относится (по названию раздела в Telegram), затем откройте соответствующий
файл в `ui/`.

## 2. Из чего состоит сообщение

Раньше сообщения собирались вручную, строка за строкой, с HTML-тегами внутри текста —
это было неудобно поддерживать. Теперь почти везде используется **набор готовых «кубиков»**
(они определены один раз в `ui/builder.py` и одинаково работают во всех файлах):

| Кубик | Что делает | Как выглядит в Telegram |
|-------|-----------|--------------------------|
| `section("Заголовок")` | Начинает новый раздел сообщения. Сам добавляет отступ (пустую строку) перед собой, если до него уже был текст. | **Заголовок** *(жирным)* |
| `line("текст")` | Обычная строка текста под заголовком. | текст |
| `bullet("пункт")` | Пункт списка. | • пункт |
| `warning("Сильный ветер")` | Предупреждение — жирный текст с ⚠️ перед ним. | ⚠️ **Сильный ветер** |
| `tip("Возьми зонт")` | Совет — жирный текст с 💡 перед ним. | 💡 **Возьми зонт** |
| `divider()` | Горизонтальная черта-разделитель. | ———————— |
| `spacer()` | Пустая строка, если нужно вставить отступ вручную. | *(пустая строка)* |

Посмотрим на реальный пример — сводку «Мой день» (файл `ui/myday.py`):

```python
b = MessageBuilder()
b.section(f"Мой день • {header} • {city}")

b.section(weather_title)       # например "☀️ Погода сегодня"
b.line(weather_line)           # например "До +22°C"

if wind_title:
    b.warning(wind_title)      # например "Сильный ветер"
    b.line(wind_line)          # например "До 12 м/с"

if word_line:
    b.section("📚 Слово дня")
    b.line(word_line)
```

В Telegram это превратится в:

```
Мой день • Четверг, 3 июля • Amsterdam

☀️ Погода сегодня
До +22°C

⚠️ Сильный ветер
До 12 м/с

📚 Слово дня
...
```

Каждый `section()` сам добавляет пустую строку перед собой — вам не нужно расставлять
отступы вручную.

## 3. Как поменять текст или эмодзи в существующем блоке

Это самый частый и самый простой вид правки — просто отредактировать строку в кавычках.

**Пример.** Хотите поменять заголовок «Мой день» на «Доброе утро»? В `ui/myday.py` находите:

```python
b.section(f"Мой день • {header} • {city}")
```

и меняете на:

```python
b.section(f"Доброе утро • {header} • {city}")
```

Точно так же можно поменять эмодзи — например, заменить 📚 на 📖:

```python
b.section("📚 Слово дня")
```
→
```python
b.section("📖 Слово дня")
```

Сохраняете файл — готово. Ничего больше трогать не нужно.

## 4. Как добавить новый блок в сообщение

Допустим, хотите добавить в «Мой день» ещё один совет после лайфхака. В `ui/myday.py`
найдите похожий существующий блок и скопируйте его структуру:

```python
if lifehack:
    b.section("💡 База знаний")
    b.line(lifehack)

# новый блок — добавили сюда
b.tip("Не забудь попить воды сегодня!")
```

Общий принцип: **найдите похожий существующий кубик в файле, скопируйте его и измените
текст**. Не изобретайте новую структуру — почти всегда нужный вам вид уже есть среди
`section` / `line` / `bullet` / `warning` / `tip` / `divider`.

## 5. Особый случай: несколько файлов используют HTML-теги напрямую

В части файлов (`food.py`, `settings.py`, `weather.py`, часть `leisure.py`, часть
`dictionary.py`) некоторые сообщения **до сих пор написаны по-старому** — текст собирается
строкой с тегами `<b>жирный</b>` и `<i>курсив</i>` внутри, а не через кубики выше. Это
сделано осознанно, не по ошибке — обычно потому, что:

- текст сообщения нужно сохранить и потом показать снова в другом месте (например избранное,
  сохранённый план поездки) — устроено проще, если это просто текст с тегами;
- в сообщение вставляется текст, сгенерированный ИИ, который сам содержит теги;
- длинный текст иногда режется на части (если не помещается в одно сообщение) — резать
  по кубикам сложнее и рискованнее, чем резать обычную строку.

**Пример** такого сообщения (карточка рецепта, файл `ui/food.py`):

```python
def food_card(data, label="Рецепт дня"):
    """HTML-текст карточки: рендерится через util.send_html (там же оживают LLM-теги в поле 'full')."""
    ...
    lines = [f"🥣 <b>{esc(label)}</b>"]
    ...
```

**Что можно менять самостоятельно в таких местах:** сам текст и эмодзи, точно так же как
в примере из раздела 3 — просто внутри строки, а не внутри `section(...)`. Например:

```python
lines = [f"🥣 <b>{esc(label)}</b>"]
```
→
```python
lines = [f"🍽️ <b>{esc(label)}</b>"]
```

**Что делать осторожно / не трогать самостоятельно:** саму структуру этих функций —
порядок сборки строк, обрезание текста по длине, места, где написано «рендерится через...»
или «персистируется как...» в комментарии над функцией. Если видите такой комментарий —
значит, у функции есть скрытая причина оставаться именно такой, и лучше попросить помощи,
прежде чем менять логику (не текст).

## 6. Как проверить, что правка не сломала сообщение

После того как поправили текст, можно посмотреть, как сообщение будет выглядеть, не запуская
самого бота. В терминале, из папки проекта:

```bash
python3 -c "
from ui import myday
msg = myday.day_summary('Четверг, 3 июля', 'Amsterdam', weather_title='☀️ Погода сегодня', weather_line='До +22°C')
print(msg.text)
"
```

Замените `myday`/`day_summary` и аргументы на файл и функцию, которую редактировали
(аргументы функции можно посмотреть прямо над её объявлением — `def имя_функции(...)`).
Команда распечатает итоговый текст сообщения — так вы сразу увидите, как оно будет выглядеть
в Telegram, без реальной отправки.

Если не уверены, какие аргументы подставить — проще всего попросить ассистента: *«покажи, как
будет выглядеть это сообщение после правки»*.

## 7. Частые ошибки

- **Не пишите `\n` внутри текста `line()`/`section()` вручную**, чтобы сделать отступ —
  кубики уже сами расставляют отступы. Лишний перенос строки внутри текста создаст двойной
  пробел между блоками.
- **Не добавляйте `spacer()` прямо перед `section()`, `warning()`, `tip()` или `divider()`** —
  они уже сами вставляют пустую строку перед собой. Если добавить `spacer()` тоже, получится
  сразу две пустые строки подряд, что выглядит как ошибка форматирования.
- **Если меняете текст в HTML-файле (раздел 5) — не забывайте про теги.** Если текст стоял
  внутри `<b>...</b>`, а вы случайно стёрли закрывающий тег `</b>` — жирным станет весь
  остаток сообщения. Проверяйте, что открывающих и закрывающих тегов поровну.
- **После правки всегда проверяйте вывод** способом из раздела 6 — это быстрее, чем
  запускать бота целиком, и сразу покажет опечатки или сломанные отступы.
