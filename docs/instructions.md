# DM Bot · Инструкция для администратора

## Главнейшее UI-правило

Во всём боте любая кнопка возврата всегда называется строго `⬅️ Назад`.
Не использовать варианты вроде `⬅️ Назад в админ`, `⬅️ Назад в систему`,
`⬅️ Назад к пользователям`, `⬅️ Назад к уведомлениям`, `⬅️ Админ`,
`⬅️ Система` или `⬅️ Пользователи`. Это правило выше остальных UX-правил.

## Что это за бот

DM Bot — личный Telegram-бот для повседневной жизни. Он помогает с:

- погодой и сводкой дня;
- гардеробом и подбором образа;
- языками: нидерландский и английский;
- досугом: фильмы, книги, музыка, концерты;
- путешествиями: страны, поездки;
- готовкой: рецепты и холодильник;
- здоровьем: ежедневный фокус, внешняя память «Мысли» и справочный вопрос врачу;
- свободным чатом с умным роутингом по функциям.

Бот работает как приватный: доступ получают только владелец и пользователи по инвайт-ссылке.

## Предпочтительный формат сообщений

Для LLM-ответов и карточек используем современное оформление через Telegram `MessageEntity`, а не HTML/Markdown-разметку в тексте.

Правила:

- первая строка — короткий заголовок без эмодзи, выделенный `MessageEntity.BOLD`;
- затем пустая строка и одно короткое предложение с сутью;
- важный общий вывод или вердикт можно оформить отдельным абзацем через `MessageEntity.BLOCKQUOTE`;
- перед списком ставится логичный заголовок по контексту, например `Что важно:`, `Почему:`, `Рекомендации:`, `Действие:`, `На что обратить внимание:`; этот заголовок выделяется `MessageEntity.BOLD`;
- любая короткая подпись перед двоеточием выделяется жирным вместе с двоеточием; в коде для этого используются `MessageBuilder.label()` и `MessageBuilder.labeled_line()`, а обычная фраза после двоеточия начинается со строчной буквы (для имён и аббревиатур регистр сохраняется явно);
- после заголовка списка нет пустой строки: список начинается сразу на следующей строке;
- пункты списка пишутся через дефис `-`, без эмодзи;
- перед финальным предложением всегда есть пустая строка;
- финальное предложение короткое, без вводок вроде `Итог:`, `Последний совет:`, `Важно:` и заканчивается точкой;
- не использовать фиксированный заголовок `Это значит:` везде — название блока должно подходить по смыслу.

Пример:

```text
Запрещено

Очень важный пункт.

The existing text and logo placement within the template must not be moved or altered.

Что важно:
- нельзя двигать логотип
- нельзя менять размер логотипа
- нельзя переносить текст

Можно менять только фон, узоры, графику и оформление.
```

## Главная логика

У бота есть 3 основных способа работы:

1. Главное меню (инлайн).
Открывается командой `/menu`, кнопкой «☰ Меню» на нижней клавиатуре или системной кнопкой меню Telegram. Разделы: `☀️ Мой день`, `👕 Гардероб`, `🥣 Готовка`, `📚 Обучение`, `🚑 Здоровье`, `✈️ Поездки`, `🍿 Досуг`, `🎚️ Настройки`.

2. Инлайн-кнопки внутри разделов.
Они запускают конкретные действия: рецепт, образ, словарь, мысли, поездки и так далее.

3. Обычный чат.
Если пользователь пишет текстом, бот пытается сам понять намерение и сразу открыть нужную функцию. Например:

- про еду — предложит рецепт;
- про фильмы — откроет рекомендации;
- про погоду — покажет прогноз;
- про мысль, тревогу или настрой — переведёт в нужный сценарий;
- если точное намерение не найдено — ответит через LLM и покажет кнопки основных разделов.

## Что может делать администратор

Раздел администратора открывается из `🎚️ Настройки` и виден только владельцу бота.

Главный экран короткий и показывает только основные входы:

- `🛠 Система`
- `👥 Пользователи`

Уведомления настраиваются в обычном разделе настроек. Отдельного админского
экрана ручных тестов уведомлений больше нет.

### 1. Система

Первый экран, который стоит проверять, если бот ведёт себя нестабильно. Все
AI-провайдеры, внешние API, Telegram и данные показаны в едином формате
`статус · сервис · раздел · статистика или проблема`. Зелёные строки не
повторяют текстовый статус, а при ограничении сразу видны затронутый раздел,
время восстановления и используемый резерв.

Сырых ошибок на главном экране нет. Кнопка `⚠️ Ошибки` открывает отдельный журнал
без кнопок копирования; `❌ Очистить ошибки` удаляет только его.

Google Books проверяется отдельным поиском одной книги не чаще раза в сутки.
Строка сервиса различает неверный ключ, отключённый API, запрет доступа, дневной
лимит, слишком частые запросы, недоступность Google и отсутствие соединения.
Фактические запросы считаются локально по дате `Europe/Amsterdam`; суточный кэш
не расходует лимит. После 1 000 запросов Google Books не вызывается, а книги
продолжают использовать Open Library.
DailyMed используется без API-ключа для официальных инструкций в «Здоровье →
Лекарства». Инструкции и выбранные разделы кэшируются на 30 дней по SET ID.
Если DailyMed не дал нужного раздела, Firecrawl ищет только по разрешённым официальным
медицинским доменам, а Tavily остаётся резервом. Для формулировки ответа вызывается
Gemini, а Groq — только последовательным резервом после реального сбоя Gemini.
В соседнем сценарии «Врач» используется тот же порядок: Gemini формирует обычный
ответ, Groq вызывается только после его сбоя. Внешний поиск там запускают только
для вопросов, требующих подтверждения, и всегда с
`include_domains` из разрешённого списка медицинских источников.
После замены ключа нажми `✨ Заменить` в разделе «Книги»: новый поиск сразу
обновит сохранённый статус, не дожидаясь следующей суточной проверки.

### 2. Пользователи

Позволяет:

- посмотреть короткую сводку по пользователям;
- создать одноразовый инвайт;
- посмотреть приветствие для нового пользователя.

Экран не показывает `user_id`, `chat_id`, последние сообщения и лишнюю активность.

### 3. Данные

Короткий статус хранилища входит в общий статус админки. Подробные ошибки нужно смотреть через `🛠 Система` → `⚠️ Ошибки`.

Отдельных вкладок `🔧 Диагностика`, `🧪 Тесты`, `☁️ API`, `🤖 LLM`, `🧠 Новости` и `⚠️ Проблемы` в обычной навигации больше нет. Кнопки из давно отправленных сообщений могут не работать: контрактом считается только текущая навигация.

## Команды

- `/menu` — открыть главное меню (инлайн); в системном меню подписана «Главное меню». Также запускает бота при первом обращении и обрабатывает инвайт-код по диплинку (`/start` остался рабочим для этого, но убран из видимого списка команд Telegram — не дублируется с `/menu`)
- `/settings` — открыть тот же экран «🎚️ Настройки», что и кнопка в главном меню; команда видна в системном меню Telegram
- `/setup` — прежний алиас настроек; остаётся рабочим, но не показывается в системном меню Telegram
- `/notes` — прежний алиас настроек; остаётся рабочим, но не показывается в системном меню Telegram
- `/admin` — прямой переход в раздел администратора; видна в системном меню только владельцу. Ручной ввод команды другим пользователем также блокируется

## Уведомления и расписание

Бот умеет слать плановые уведомления по времени `Europe/Amsterdam`.

Текущие активные расписания (сокращены с 13 до 6 основных + отдельное экстренное
погодное предупреждение — часть старых уведомлений дублировала друг друга и
приходила по 5 сообщений подряд по утрам):

- ежедневно в `08:30` — «Утро» (Мой день + погода)
- ежедневно в `08:45` — погодное предупреждение (экстренное, только если есть значимый фактор)
- по пятницам в `09:50` — прогрев кэша концертов (служебное, без сообщения пользователю)
- по пятницам в `10:00` — «Куда сходить» (афиша недели + новые концерты любимых артистов)
- ежедневно в `11:00` — «Практика языка» (слово дня по активному языку + живая фраза)
- ежедневно в `14:00` — «😮‍💨 Есть что выгрузить?»
- ежедневно в `20:30` — прогноз на завтра
- ежедневно в `21:00` — «😌 Закроем день», только если остались открытые мысли

Включение и выключение делается в разделе `🎚️ Настройки` → `Уведомления`.

## Доступ и инвайты

Бот приватный.

Как это работает:

- владелец задаётся через `CHAT_ID`;
- новые пользователи получают доступ только через инвайт;
- инвайт создаётся через админский экран;
- после активации инвайт одноразовый;
- доступ можно отозвать из админки.

Если пользователь пишет боту без доступа, бот его не впускает.

## Провайдеры LLM

Бот умеет работать с несколькими провайдерами:

- Gemini
- Groq
- Cohere
- GitHub Models
- Cloudflare
- OpenRouter — аварийный fallback для публичного текста и явно разрешённых JSON-сценариев, например рецептов в «Готовке»

Они используются не одинаково:

- часть задач идёт через быстрые и дешёвые модели;
- часть — через более "умные";
- часть сценариев принудительно направлена в конкретного провайдера.

Это сделано специально, чтобы балансировать качество, скорость и стоимость.

Для Готовки действует отдельный фиксированный порядок: `Gemini → Groq → GitHub Models → OpenRouter`. Если все четыре недоступны, рецепт форматируется кодом напрямую из ответа Spoonacular, без AI.

Cohere Trial ограничен локальным месячным счётчиком на 1 000 фактических
запросов по времени `Europe/Amsterdam`. Ответы из AI-кэша не учитываются. После
исчерпания лимита Cohere не вызывается, а запрос сразу переходит в GitHub Models.

Экран `🛠 Система` не подставляет справочные или демонстрационные тарифы. Он
показывает квоту из ответа API и rate-limit-заголовков, локальный счётчик для
Google Books и Cohere либо фактический расход за текущий период. Для Gemini
лимит задаётся только явно через `GEMINI_DAILY_LIMIT`; без него показываются
число запросов сегодня и выбранная модель.
Фактический запрос и фоновая проверка обновляют одно состояние провайдера, поэтому
активный резерв, квота и восстановление не расходятся между экранами.

## Что хранит бот

Внутри сохраняются:

- настройки города и личные предпочтения;
- словарь и темы обучения;
- гардероб;
- холодильник и рецепты;
- любимое и единый внутренний стоп-лист «Не рекомендовать» — фильмы, книги, музыка, страны;
- сохранённые заметки;
- исходные записи раздела «😮‍💨 Мысли», их скрытая классификация и история завершённых общих разборов;
- лог ошибок и история LLM-расходов.

Если бот перезапускается, временные сценарные состояния могут сброситься:

- незавершённый ввод;
- игровые состояния;
- промежуточная история некоторых интерактивных веток.

Это нормальное поведение.

## Что проверять, если что-то сломалось

Базовый порядок диагностики:

1. Открой `🎚️ Настройки` → админский раздел → `🛠 Система`.
2. Проверь строку `Ответы`, затем строки сервисов и используемые резервы.
3. Если проблема в уведомлении — проверь его настройку и свежую запись в `⚠️ Ошибки`.
4. Если подозрение на LLM или внешний сервис — смотри строку этого сервиса в `🛠 Система`.
5. Если проблема у конкретного пользователя — проверь его доступ в `👥 Пользователи`.
6. Если нужна свежая ошибка целиком — открой `🛠 Система` → `⚠️ Ошибки`.

Если бот отвечает странно в чате, но разделы работают нормально, чаще всего проблема не в Telegram, а в маршрутизации текста или в конкретном LLM-провайдере.

Обычный AI-ответ имеет общий лимит 10 секунд, сложная рекомендация — 15 секунд.
Лимит включает основной сервис и резервы. После него бот использует допустимый
кэш или локальный fallback. Служебные аудиты выполняются после запуска polling,
а фоновый прогрев уступает активному пользовательскому действию.

## Что нужно для запуска

Строго обязательные переменные окружения (бот не запустится без них):

- `TELEGRAM_TOKEN`
- `GEMINI_API_KEY`

Все остальные переменные опциональны — без них бот запускается и работает, но соответствующая функция или провайдер будут недоступны:

- `CHAT_ID` — без него не определён владелец бота;
- `DATABASE_URL` — без него хранилище работает в памяти (данные не переживают перезапуск);
- `COHERE_API_KEY` — основной провайдер для языковых, классификационных и других структурированных задач; Trial ограничен локальным месячным счётчиком на 1 000 запросов, после лимита используется GitHub Models;
- `COHERE_MODEL` — модель Cohere, по умолчанию `command-a-plus-05-2026`;
- `GEMINI_MODEL` — модель Gemini, по умолчанию `gemini-2.5-flash`;
- `GEMINI_DAILY_LIMIT` — подтверждённая дневная квота проекта Gemini; если не задана или равна `0`, админка показывает только фактическое число запросов;
- `GITHUB_MODELS_TOKEN` — универсальный AI-резерв через GitHub Models; токену нужен доступ `models:read`;
- `GITHUB_MODELS_MODEL` — модель GitHub Models, по умолчанию `openai/gpt-4.1-mini`;
- `GOOGLE_BOOKS_API_KEY` — поиск и проверка метаданных и обложек книг в Google Books; доступ проверяется официальным поисковым запросом одной книги раз в сутки, фактические запросы ограничены локальным дневным счётчиком на 1 000;
- `LANGUAGETOOL_API_URL` — адрес LanguageTool для проверки письменных ответов тренажёра и учебных данных; по умолчанию `https://api.languagetool.org/v2`;
- `SPOONACULAR_API_KEY` — основной источник рецептов и винных сочетаний для Готовки;
- `THEMEALDB_API_KEY` — резервный источник базовых рецептов для Готовки; по умолчанию используется публичный ключ `1`;
- `AZURE_SPEECH_KEY` — ключ Azure AI Speech для озвучивания нидерландских слов и примеров;
- `AZURE_SPEECH_REGION` — регион ресурса Azure Speech, например `westeurope`;
- `AZURE_SPEECH_VOICE` — нидерландский голос; по умолчанию `nl-NL-MaartenNeural`;
- `AZURE_SPEECH_RATE` — скорость произношения; по умолчанию `-10%`;
- `OPENROUTER_API_KEY`, `GROQ_API_KEY`, `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID` — остальные резервные LLM-провайдеры;
- `TMDB_API_KEY`, `TICKETMASTER_API_KEY`, `TAVILY_API_KEY`, `FIRECRAWL_API_KEY` — фильмы, концерты, веб-поиск;
- `RESTCOUNTRIES_API_KEY` — уточнение кода и русского названия страны для общей карточки в разделе «Поездки»;
- `PEXELS_API_KEY` — фотографии для карточек рецептов;
- `ZEROENTROPY_API_KEY` — вспомогательный rerank-сервис;
- `OPENROUTER_MODEL` — модель OpenRouter fallback.

Актуальное состояние ключей всегда можно сверить в `🎚️ Настройки` → админский раздел → `📊 Система`.

## Коротко

Если нужен самый практичный админский сценарий:

1. `🎚️ Настройки` → `Администратор`
2. смотри `🛠 Система`
3. проверяй, какой резерв используется при ограничениях
4. проверяй настройки уведомлений в обычных `Настройках`
5. управляй доступом через `👥 Пользователи`
6. смотри свежие ошибки в `🛠 Система` → `⚠️ Ошибки`

Этого достаточно, чтобы контролировать бота без погружения в код.
