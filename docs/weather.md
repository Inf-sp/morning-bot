# Погода

## Назначение
Раздел "Погода" показывает прогноз для города пользователя — на сегодня, на завтра, подробный почасовой прогноз и на ближайшую неделю. Помогает быстро понять, что надевать и чего ожидать от дня, без необходимости открывать сторонние приложения.

## Как это работает
Погодные данные бот берёт из OpenWeatherMap One Call API 4.0 по ключу `WEATHER_API_KEY`. Ключ создаётся в кабинете OpenWeatherMap: https://home.openweathermap.org/api_keys, а на One Call API 4.0 должна быть отдельно активирована подписка на https://home.openweathermap.org/subscriptions — без неё запросы будут отклоняться с ошибкой авторизации. Утром бот заранее прогревает прогноз перед "Мой день", а успешный ответ сохраняет в устойчивый кэш на 3 часа: повторные разделы вроде погодного предупреждения или гардероба читают эти же данные без новых запросов, а прогноз при этом чаще обновляется свежими данными. Если OpenWeather временно недоступен, бот может использовать сегодняшний закэшированный прогноз вместо пустого блока.
При явном открытии «Моего дня» текущие условия уточняются отдельно и кэшируются
на 10 минут; дневной и почасовой прогноз повторно не запрашиваются.

One Call API 4.0 разбит на несколько отдельных запросов (текущая погода, почасовой прогноз, дневной прогноз), которые бот делает параллельно, чтобы не терять скорость ответа. Погодные предупреждения (алерты) в этой версии API сначала приходят как краткий идентификатор, и только если алерт действительно есть — бот делает один дополнительный запрос за его деталями.

В зависимости от того, что выбрал пользователь, бот показывает разные форматы:

- **Погода на сегодня** — заголовок с датой и городом, основная строка с температурой, дождём (если он есть и вероятность высокая) и ветром. Дополнительно бот подмешивает шутливый факт о погоде в одном из городов мира прямо сейчас (например, "кстати, сегодня в Дубае +45°C").
- **Погода на завтра** — похожий формат, но вместо шутки о мировой погоде бот показывает предупреждение об опасной погоде, если ожидается шторм, сильный ветер, снег или ливень.
- **Погода на завтра (упрощённый вариант)** — по сути то же самое, что "на завтра", но вместо метеофакта показывается короткий текстовый итог-обзор погоды на 2-3 предложения (без слова "зонт" — специально избегается).
- **Подробный прогноз на сегодня** — прогноз разбит на части дня (утро/день/вечер), для каждой части — своя температура, дождь, ветер. В конце добавляется дерзкая шутка с советом по одежде (нужна ли куртка или зонт).
- **Прогноз на неделю** — жирный заголовок с периодом и городом, диапазон дневных температур за семь дней без ночных минимумов, одна короткая характеристика недели и семь строк с максимальной дневной температурой. В конце показывается одно практическое предложение для прогулок, велосипеда или поездок. Дни не объединяются, длинные описания и отдельный «Метео-итог» не используются.

Недельный экран целиком строится по данным OpenWeatherMap. Краткое описание и
совет выбираются локальными правилами по температуре, осадкам и ветру, поэтому
бот не придумывает погодные условия и не вызывает AI для этого прогноза.

Отдельно есть ежедневное **погодное предупреждение** в 08:45. Оно приходит только в дни, когда ожидается хотя бы один значимый фактор: гроза, гололёд, штормовой ветер, сильный дождь, жара, высокий UV, снег, мороз, туман или высокая влажность. Спокойные дни не присылаются.

В 20:30 уведомление с прогнозом на завтра начинается с компактного заголовка
`Завтра · Сб, 18 июля · Алкмар 📍`. В основной строке ветер показывается без
эмодзи и с понятной классификацией, например `Ветер 7 м/с · умеренный`.

Формат предупреждения всегда компактный:

```
⚠️ Погодное предупреждение

<1-3 самых важных погодных события>

🕒 Когда:
<интервал времени или период>

🎒 Что сделать:
<2-4 короткие рекомендации>
```

Если сработало несколько факторов, бот объединяет их в одно сообщение: выбирает до трёх самых важных событий и до четырёх неповторяющихся советов. Содержание учитывает персональные данные, где они есть: например, велосипедные советы при ветре/дожде/тумане и подсказку про дождевик, если в гардеробе есть непромокаемая верхняя одежда.

При смене города (текстом, через геолокацию или подтверждением при первой настройке) бот заново определяет координаты через OpenWeatherMap Geocoding API по тому же `WEATHER_API_KEY`; при неудаче использует запасной Nominatim (OpenStreetMap). Бот также старается распознать неофициальные названия городов ("Питер" → "Санкт-Петербург") и подобрать правильное русское написание, сверяясь через модель ИИ. После успешной смены города бот сразу обновляет и присылает актуальную сводку "Мой день".

При отправке геолокации (через кнопку "поделиться местоположением" в Telegram) город определяется автоматически по координатам через отдельный сервис обратного геокодирования, после чего сразу показывается погода на сегодня.

## Основные возможности
- Прогноз на сегодня, на завтра, подробный почасовой прогноз на сегодня и на неделю вперёд.
- Показ температуры, вероятности и силы дождя, силы и направления ветра.
- Предупреждения об опасной погоде: сильный ветер (шквалы), снегопад, ливень или гроза — с более развёрнутыми предупреждениями для Нидерландов (упоминание задержек поездов NS, ссылка на Buienradar).
- Ежедневное погодное предупреждение по расписанию, только если есть значимый погодный риск.
- Пояснение о комфорте с учётом влажности воздуха (например, что жара из-за влажности ощущается сильнее, чем показывает термометр).
- Интересные факты: шутливое сравнение с погодой в другом городе мира.
- Смена города текстом (с распознаванием неофициальных названий) или через геолокацию.
- Кнопки быстрого перехода между разделами (назад к сводке дня, к недельному прогнозу).

## Настройки
- **Город и координаты** — главная настройка, от неё зависит вообще весь прогноз; меняется через ввод названия города или отправку геолокации.
- **Погодное предупреждение** — отдельное уведомление в настройках уведомлений; можно включить или выключить.
- Явных пользовательских настроек (включить/выключить какие-то блоки погоды, изменить единицы измерения и т.п.) в разделе не предусмотрено — прогноз показывается в едином формате для всех.

## Ограничения
- Дождь в сводке упоминается (эмодзи, проценты, фраза "дождь") только если вероятность осадков составляет не менее 50%; более низкая вероятность считается несущественной и не показывается вовсе.
- В «Моём дне» вероятность и объём осадков уточняются по почасовому окну 08:00–22:00, чтобы морось днём или дождь вечером не пропадали из-за суточного агрегата.
- Текущий код OpenWeather имеет приоритет для осадков: если дождь уже начался,
  «Мой день» пишет «Дождь сейчас» без выдуманного процента.
- Даже при высокой вероятности дождь может быть проигнорирован, если ожидаемое количество осадков в миллиметрах ничтожно мало (меньше десятой доли миллиметра) — вероятность может быть "статистическим шумом".
- Предупреждение о ветре с усиленной формулировкой ("сильный ветер"/"очень сильный ветер") появляется только начиная с определённой силы ветра; более слабый ветер описывается нейтрально.
- Штормовое предупреждение (шквалы, ливень, снег) показывается только при действительно серьёзных значениях: ветер выше особого порога (около 15 м/с), сильные коды осадков (ливень, гроза) или выпадение от 15 мм осадков за день и больше.
- Расширенные предупреждения про поезда NS и сервис Buienradar показываются только для Нидерландов; для других стран — общие формулировки про задержки транспорта.
- Пояснение про влажность воздуха показывается не всегда, а только при определённых сочетаниях влажности и температуры (например, высокая влажность вместе с достаточно высокой температурой, либо очень низкая влажность).
- Кеш погодных данных живёт 3 часа. Если свежий запрос не прошёл, бот может fallback'нуться на сегодняшний кэш до 24 часов, но не подставляет вчерашний прогноз как сегодняшний.
- Дневной лимит запросов к OpenWeatherMap выставлен на уровне 1000 запросов/день (в кабинете OpenWeather) — это реальный бесплатный потолок One Call API 4.0, а не искусственно заниженное значение.
- Если город не удаётся найти ни через один из геосервисов, бот сообщает, что не нашёл город, и просит попробовать снова.
- Погодное предупреждение не учитывает AQI и пыльцу: для этого нужен отдельный источник данных. Переключатель аллергии заведён на будущее, но без данных о пыльце фактор честно не срабатывает.
- Интервал «Когда» в погодном предупреждении строится по дневному окну; ночные явления могут не попасть в утреннее сообщение.

## Что можно улучшить
- Добавить пользовательский выбор единиц измерения (например, температура в Фаренгейтах для части аудитории).
- Дать возможность подписаться на отдельные уведомления при штормовых предупреждениях, не дожидаясь захода в раздел.
- Показывать историю смены городов, чтобы можно было быстро переключаться между несколькими "своими" городами.
- Расширить список стран с локализованными предупреждениями (сейчас развёрнутый формат только для Нидерландов).
- Подключить AQI и пыльцу для погодных предупреждений.
