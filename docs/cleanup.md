# Чистка

## Назначение

Единый экран для просмотра и массового удаления записей из любого списка бота — сохранённого, любимого, скрытого, гардероба, словаря, холодильника, рецептов, дневника здоровья и так далее. Вместо того чтобы у каждого раздела был свой самодельный способ «отметить и удалить», все они используют один и тот же движок: пользователь видит одинаковый паттерн взаимодействия независимо от того, чистит он словарь нидерландского или список продуктов в холодильнике.

## Как это работает

### Вход в чистку

Экран чистки открывается кнопкой внутри соответствующего раздела — «❌ Удалить» в Любимом, «🗑 Удалить» в Дневнике тревог, «Настройки гардероба» → выбор зоны и подкатегории и т.д. Своей отдельной точки входа в меню у «Чистки» нет — это всегда действие внутри чужого раздела.

При открытии показывается: заголовок коллекции, счётчик «Всего: N · отмечено: M», список элементов постранично (до 8 на страницу) с чекбоксами (▫️ / ✅), и подсказка, что делать дальше.

### Отметка и постраничная навигация

Нажатие на элемент переключает его отметку (без перезагрузки всего списка — редактируется то же сообщение). Если элементов больше одной страницы, снизу появляются кнопки «◀️» / «N/M» / «▶️» для перелистывания; отметки, сделанные на других страницах, не сбрасываются при переходе.

Когда на текущей странице два и более элемента, появляется кнопка «✅ Выбрать все на странице» — отмечает все элементы страницы разом; при повторном нажатии (если страница уже полностью отмечена) кнопка меняет текст на «✅ Снять выбор на странице» и снимает отметки. Это касается только видимой страницы, не всей коллекции.

### Групповое действие и подтверждение

Как только отмечен хотя бы один элемент, появляется кнопка финального действия — её текст объясняет последствие, а не техническую операцию:

| Коллекция | Текст кнопки | Подтверждение |
|---|---|---|
| Любимое (фильмы/книги/музыка/страны) | «Убрать из любимого» | Не требуется |
| Скрытое (чёрный список) | «Вернуть в рекомендации» | Не требуется |
| Сохранённое | «Удалить сохранённое» | Требуется |
| Просмотренное / Прочитанное | «Убрать из просмотренного» | Требуется |
| Гардероб | «Удалить вещи» | Требуется |
| Рецепты | «Удалить рецепты» | Требуется |
| Словарь | «Удалить слова» / «Удалить фразы» | Требуется |
| Холодильник | «Удалить продукты» | Требуется |
| Здоровье (Лагом) | «Удалить принципы» | Требуется |

Разница в подтверждении не случайна: «Убрать из любимого» и «Вернуть в рекомендации» — обратимые действия (можно добавить обратно или элемент сам вернётся в рекомендации), поэтому удаление происходит сразу по нажатию. Все остальные коллекции стираются без возможности восстановления — для них первое нажатие кнопки показывает отдельный экран:

```text
Удалить 3? Это действие нельзя отменить.

[🗑 Удалить 3] [⬅️ Назад]
```

Реальное удаление происходит только после повторного явного нажатия «🗑 Удалить N». `⬅️ Назад` возвращает к обычному экрану чистки без изменений.

### Добавление прямо с экрана чистки

Для Любимого (фильмы, страны, артисты, книги) на экране чистки есть также кнопка «✏️ Добавить …» — можно пополнить список, не выходя в другой раздел. У Скрытого такой кнопки нет: чёрный список пополняется автоматически, когда пользователь отмечает «Не нравится»/«Заменить» на карточке рекомендации, а не вручную.

### Защита от гонки данных

Если список изменился в другом месте параллельно с тем, как открыт экран чистки (например, элемент удалили из другого раздела или другого устройства), нажатие финального действия не выполняется вслепую — вместо этого показывается:

```text
Список уже изменился. Откройте его заново.
```

Экран нужно открыть заново — так исключается ситуация, когда случайно удаляется не тот элемент из-за того, что список успел перестроиться. Открытый, но забытый экран чистки автоматически считается устаревшим через сутки — тоже потребует переоткрытия.

## Связи с другими разделами

- **Любимое / Скрытое** (Досуг → Кино/Книги/Музыка, Путешествия) — чистка вызывается из настроек соответствующей категории.
- **Гардероб** — удаление вещей идёт через ту же чистку, но с дополнительной иерархией Зона → Подкатегория перед списком конкретных вещей (см. `docs/wardrobe.md`).
- **Обучение → Словарь** — удаление слов и фраз идёт через общий экран чистки; битые записи должны отсекаться при добавлении, а не проверяться отдельной кнопкой.
- **Готовка** — холодильник и сохранённые рецепты чистятся тем же способом.
- **Здоровье** — Лагом (личные принципы) использует тот же экран.
- **Настройки → Сохранённое** — заметки, сохранённые кнопкой «Сохранить» под ответом бота, чистятся здесь же, с делением по категориям.

## Основные возможности

- Единый экран отметки и массового удаления для всех списков бота — одинаковое поведение независимо от раздела.
- Постраничный просмотр (8 элементов на страницу) с сохранением отметок между страницами.
- «Выбрать/Снять все на странице» одной кнопкой.
- Контекстные тексты финального действия, которые называют последствие, а не техническую операцию.
- Экран подтверждения перед необратимым удалением — реальное стирание требует повторного явного нажатия.
- Обратимые действия (Любимое, Скрытое) выполняются сразу, без лишнего подтверждающего шага.
- Защита от гонки данных: если список изменился параллельно, ничего не удаляется вслепую, экран предлагает переоткрыться.
- Кнопка добавления элемента прямо с экрана чистки — для категорий Любимого.

## Настройки

У раздела нет собственных настроек — поведение (какие тексты, требуется ли подтверждение, доступна ли кнопка добавления) целиком определяется тем, из какого раздела бота вызвана чистка.

## Ограничения

- Для больших коллекций (например, сохранённого с сотнями заметок) массовый выбор ограничен постраничным «Выбрать все на странице».
- Открытый экран чистки живёт 24 часа — после этого при попытке что-то удалить нужно открыть его заново, даже если список не менялся.
- Список стран/артистов/книг, доступный из карточек рекомендаций (легаси-путь настроек), использует более старую позиционную адресацию элементов и не входит в описанную здесь защиту от гонки данных — судьба этого пути ждёт отдельного решения.
- Записи «Истории самочувствия» показывают только дату и краткий итог разбора тревог — детальный список самих тревог в этом экране не отображается.

## Что можно улучшить

- Добавить отдельный массовый выбор всей коллекции для разделов, где список часто вырастает большим (сохранённое, дневник).
- Единая адресация элементов (без деления на «старый позиционный формат» и «новый формат по id») для всех коллекций без исключений.
- Групповое перемещение элементов между коллекциями (например, из Просмотренного сразу в Скрытое) без отдельного захода в исходный раздел.
