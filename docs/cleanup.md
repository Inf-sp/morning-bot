# Чистка

## Назначение

Единый экран для просмотра и массового удаления записей из пользовательских списков бота — сохранённого, любимого, гардероба, холодильника, рецептов, дневника здоровья и так далее. Вместо того чтобы у каждого раздела был свой самодельный способ «отметить и удалить», все они используют один и тот же движок: пользователь видит одинаковый паттерн взаимодействия независимо от того, чистит он список продуктов в холодильнике или сохранённое. Удалённые фильмы, книги, артисты и страны попадают во внутреннюю категорию «Не рекомендовать»; отдельных кнопок «Скрытое» и «Смотрел» в категориях нет.

Словарь — исключение: там вместо группового выбора используется тап по слову (список без перевода → карточка слова → «Удалить»), см. `docs/dictionary.md`.

## Как это работает

### Вход в чистку

Экран чистки открывается кнопкой внутри соответствующего раздела — любое удаляющее действие начинается с единого эмодзи `❌`: «❌ Удалить», «❌ Убрать из любимого» и так далее. Для удаления из Гардероба сначала открывается карточка вещи; в ней нет исправления, только удаление с отдельным подтверждением. Своей отдельной точки входа в меню у «Чистки» нет — это всегда действие внутри чужого раздела.

При открытии показывается обычный список элементов постранично (до 8 на страницу), без чекбоксов, и кнопка «✏️ Изменить». После её нажатия появляются чекбоксы (□ / ✅) и массовые действия.

### Отметка и постраничная навигация

В режиме редактирования нажатие на элемент переключает его отметку (без перезагрузки всего списка — редактируется то же сообщение). Кнопка «Готово» возвращает обычный просмотр. Если элементов больше одной страницы, снизу появляются кнопки «◀️» / «N/M» / «▶️» для перелистывания; отметки, сделанные на других страницах, не сбрасываются при переходе.

Когда на текущей странице два и более элемента, непосредственно над навигацией появляется кнопка «*️⃣ Выбрать все на странице» — отмечает все элементы страницы разом; при повторном нажатии (если страница уже полностью отмечена) кнопка меняет текст на «✅ Снять выбор на странице» и снимает отметки. Это касается только видимой страницы, не всей коллекции.

### Групповое действие и подтверждение

Как только отмечен хотя бы один элемент, появляется кнопка финального действия — её текст объясняет последствие, а не техническую операцию:

| Коллекция | Текст кнопки | Подтверждение |
|---|---|---|
| Любимое (фильмы/книги/музыка/страны) | «❌ Убрать из любимого» | Не требуется |
| Сохранённое | «❌ Удалить сохранённое» | Требуется |
| Просмотренное / Прочитанное | «❌ Убрать из просмотренного» | Требуется |
| Гардероб | «❌ Удалить вещи» | Требуется |
| Рецепты | «❌ Удалить рецепты» | Требуется |
| Холодильник | «❌ Удалить продукты» | Требуется |

«Убрать из любимого» выполняется сразу, потому что элемент можно добавить обратно. Все остальные пользовательские коллекции стираются без возможности восстановления — для них первое нажатие кнопки показывает отдельный экран:

```text
Удалить 3? Это действие нельзя отменить.

[❌ Удалить 3] [⬅️ Назад]
```

Реальное удаление происходит только после повторного явного нажатия «❌ Удалить N». `⬅️ Назад` возвращает к обычному экрану чистки без изменений.

### Добавление прямо с экрана чистки

Для Любимого (фильмы, страны, артисты, книги) на экране чистки первой строкой есть кнопка «🆕 Добавить объект» с конкретным названием объекта — можно пополнить список, не выходя в другой раздел.

### Защита от гонки данных

Если список изменился в другом месте параллельно с тем, как открыт экран чистки (например, элемент удалили из другого раздела или другого устройства), нажатие финального действия не выполняется вслепую — вместо этого показывается:

```text
Список уже изменился. Откройте его заново.
```

Экран нужно открыть заново — так исключается ситуация, когда случайно удаляется не тот элемент из-за того, что список успел перестроиться. Открытый, но забытый экран чистки автоматически считается устаревшим через сутки — тоже потребует переоткрытия.

## Связи с другими разделами

- **Любимое** (Досуг → Кино/Книги/Музыка, Поездки) — чистка вызывается из настроек соответствующей категории. Кнопок для внутренних списков исключений нет.
- Удаление из рекомендательных списков не возвращает элемент в ротацию: он записывается в общий стоп-лист «Не рекомендовать».
- **Гардероб** — категория открывает плоский список вещей, затем карточку конкретной вещи с «❌ Удалить» и обязательным подтверждением (см. `docs/wardrobe.md`).
- **Готовка** — «❌ Удалить продукты» открывается внутри категории холодильника и показывает только её продукты. На главном экране «Мой холодильник» общей кнопки удаления нет. Сохранённые рецепты чистятся тем же способом.
- **Настройки → Сохранённое** — заметки, сохранённые кнопкой «Сохранить» под ответом бота, чистятся здесь же, с делением по категориям.

## Основные возможности

- Единый экран отметки и массового удаления для всех списков бота — одинаковое поведение независимо от раздела.
- Постраничный просмотр (8 элементов на страницу) с сохранением отметок между страницами.
- «*️⃣ Выбрать все на странице / ✅ Снять выбор на странице» одной кнопкой.
- Контекстные тексты финального действия, которые называют последствие, а не техническую операцию.
- Экран подтверждения перед необратимым удалением — реальное стирание требует повторного явного нажатия.
- Обратимое действие «Убрать из любимого» выполняется сразу, без лишнего подтверждающего шага.
- Защита от гонки данных: если список изменился параллельно, ничего не удаляется вслепую, экран предлагает переоткрыться.
- Кнопка добавления элемента прямо с экрана чистки — для категорий Любимого.

## Настройки

У раздела нет собственных настроек — поведение (какие тексты, требуется ли подтверждение, доступна ли кнопка добавления) целиком определяется тем, из какого раздела бота вызвана чистка.

## Ограничения

- Для больших коллекций (например, сохранённого с сотнями заметок) массовый выбор ограничен постраничным «*️⃣ Выбрать все на странице».
- Открытый экран чистки живёт 24 часа — после этого при попытке что-то удалить нужно открыть его заново, даже если список не менялся.
- Записи «Истории самочувствия» показывают только дату и краткий итог разбора тревог — детальный список самих тревог в этом экране не отображается.

## Что можно улучшить

- Добавить отдельный массовый выбор всей коллекции для разделов, где список часто вырастает большим (сохранённое, дневник).
- Единая адресация элементов (без деления на «старый позиционный формат» и «новый формат по id») для всех коллекций без исключений.
- Групповое перемещение элементов между коллекциями (например, из Просмотренного сразу в Скрытое) без отдельного захода в исходный раздел.
