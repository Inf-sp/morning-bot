# Сохранённое

## Назначение
Это сквозная механика, которая работает под ответами бота почти в любом разделе — кино, книги, музыка, путешествия, еда, гардероб, здоровье и другое. Она позволяет отметить понравившийся ответ бота кнопкой, а потом решить, что с ним делать дальше: оставить как заметку на память, перевести в постоянные любимые (чтобы бот учитывал вкус) или скрыть навсегда.

## Как это работает
Под многими ответами бота (рекомендация фильма, книги, музыки, идея для поездки и так далее) есть кнопка «💾 Сохранить». После нажатия она меняется на «✅ Сохранено», а текст ответа целиком попадает в личный список «Сохранённые». Повторное нажатие снимает сохранение и возвращает подпись «💾 Сохранить». Карточка при этом остаётся на месте. Это буфер «на потом», он не влияет на вкус и на будущие рекомендации.

Для кино, книг, музыки и путешествий на самой карточке рекомендации рядом с «💾 Сохранить» есть ещё прямые действия, доступные без захода в «Сохранённые»:

- **❤️ В любимые** — элемент сразу переходит в постоянный список любимых по своей категории. Посещённые страны находятся прямо в «Поездки → Мои страны».
- **✅ Уже видел/читал/знаю** — элемент запоминается как знакомый (не повторять), но вкус не меняется — в отличие от «В любимые», это нейтральная пометка.

Дальше с каждым сохранённым элементом (из «Сохранённых») можно поступить одним из трёх способов:

- **В любимые** — элемент переходит в постоянный список любимых по своей категории и убирается из «Сохранённых». С этого момента бот учитывает его при подборе новых рекомендаций — то есть это не просто список для просмотра, а данные, на которые бот опирается, чтобы точнее понимать вкус.
- **Скрыть** — элемент убирается из «Сохранённых» и попадает в скрытый список («чёрный список») своей категории. Бот запоминает, что этот фильм, книга, артист или страна больше не интересны, и никогда не предложит их снова.
- **Просто удалить** — элемент убирается без каких-либо последствий: не влияет на рекомендации и не блокирует повторные предложения.

Скрытые, просмотренные/прочитанные/знакомые и удалённые из рекомендательных списков элементы попадают в одну внутреннюю категорию «Не рекомендовать». Она не относится отдельно к Кино, Книгам или Музыке: тип объекта хранится только для корректной фильтрации. Отдельной кнопки и пользовательского экрана этого стоп-листа в категориях и списках «Любимое» нет.

Категория элемента определяется автоматически по тому, из какого раздела бота он был сохранён:
- **Кино** — фильмы и сериалы.
- **Книги** — книги и списки к прочтению.
- **Музыка** — артисты, музыка и концерты.
- **Поездки** — страны и поездки.
- **Еда** — рецепты и питание.
- **Гардероб** — образы и покупки.
- **Здоровье** — темы про здоровье, мотивацию, баланс.
- **Прочее** — всё остальное, что не попало ни в одну из перечисленных категорий (например, отдельные заметки из общего чата с ассистентом).

Только для первых четырёх категорий (кино, книги, музыка, путешествия) бот понимает, куда именно перевести элемент в «любимое» или в какой список «скрытых» его добавить — поэтому у карточек с такой категорией есть кнопки «В любимые» и «Скрыть». Для остальных категорий (еда, гардероб, здоровье, прочее) такой логики пока нет, и карточка предлагает только удаление.

Список «Сохранённые» открывается кнопкой «⭐️ Сохранённое» с главного экрана Настроек (рядом с «❤️ Любимое») и разбит по категориям — можно зайти в конкретную категорию (например, «Кино») и увидеть все сохранённые туда карточки. Внутри карточки видно исходный текст ответа бота, дату сохранения и категорию.

## Основные возможности
- Сохранение любого понравившегося ответа бота кнопкой «💾 Сохранить» с возможностью снять сохранение повторным нажатием — не влияет на вкус.
- Прямые кнопки «❤️ В любимые» и «✅ Уже видел/читал/знаю» на карточках кино, книг, музыки и путешествий — минуя «Сохранённые».
- Просмотр списка сохранённого, разбитого по категориям, с количеством элементов в каждой.
- Просмотр отдельной карточки сохранённого элемента с полным текстом.
- Перевод элемента из «Сохранённых» в постоянные любимые — для категорий, которые бот умеет распознавать (кино, книги, музыка, путешествия). Любимые в дальнейшем влияют на рекомендации.
- Скрытие элемента — бот больше никогда не предложит тот же фильм, книгу, артиста или страну.
- Удаление фильма, книги, артиста или страны из сохранённого либо любимого также переносит элемент в «Не рекомендовать», чтобы удалённая рекомендация не вернулась снова.
- Простое удаление элемента без каких-либо последствий для будущих рекомендаций.
- Массовая очистка сохранённого по отдельной категории или полностью.
- Экспорт всех сохранений и любимого в файл.

## Настройки
Отдельного экрана настроек у этой механики нет — управление происходит прямо в списке «Сохранённые»: можно зайти в любую категорию, посмотреть, что там накопилось, и почистить её — как удалить один элемент, так и очистить сразу всю категорию или весь список целиком. Списки любимого (по кино, книгам, музыке, странам) также доступны для просмотра и очистки из настроек бота; отдельные списки «Скрытое» рядом с ними не показываются.

## Ограничения
- Кнопки «В любимые» и «Скрыть» показываются только для элементов с понятной категорией — кино, книги, музыка, путешествия. Для заметок без чёткой категории (например, общих сообщений из чата с ассистентом, а также еды, гардероба и здоровья) доступно только простое удаление.
- «Сохранённые» — это временное хранилище: пока элемент не переведён в «любимое» или не скрыт, он не влияет на рекомендации бота.
- Перевод в любимое необратим через интерфейс: чтобы убрать элемент из любимого, нужно зайти в сам список «Любимое» и удалить его оттуда явно.

## Что можно улучшить
- Добавить логику «любимое» и «чёрный список» для оставшихся категорий — еды, гардероба и здоровья.
- Показывать в карточке любимого, сколько раз бот уже использовал его при подборе рекомендаций.
- Разрешить массово переносить несколько сохранённых элементов в любимые за один раз, а не по одному.
