# Админ

Роль: администратор бота, владелец `config.CHAT_ID`.

Цель панели: за несколько секунд ответить на четыре вопроса:

- всё работает?
- есть ли проблемы?
- кто пользуется ботом?
- как быстро проверить вручную?

## Навигация

```text
🛠 Админ
├── 📊 Система
│   ├── 🔧 Диагностика
│   │   ├── ☁️ API
│   │   ├── 🤖 LLM
│   │   ├── 🧠 Новости
│   │   └── 📜 Логи
│   └── 📜 Логи
├── 🔔 Уведомления
├── 👥 Пользователи
└── 🧪 Тесты
```

Главный экран показывает только короткую сводку: общий статус, систему,
уведомления, пользователей, данные и время обновления. На нём нет расписания,
следующего уведомления, Tavily, Personal News, OpenWeather-остатков, LLM-токенов,
последних API-вызовов, rate-limit логов и истории OK-событий.

## 📊 Система

Единая сводка по модулям и внешним сервисам. Одна строка - один сервис или модуль:

- LLM
- OpenWeather
- Telegram
- TMDB
- Pexels
- Cloudflare
- Groq
- Gemini
- Tavily
- Новости
- Фото рецептов
- Данные
- Планировщик

Кнопка `🔄 Проверить систему` использует существующие probe-функции и показывает
результат на этом же экране. Подробные лимиты, ошибки, кэш и внутренние метрики
не показываются здесь; для этого есть `🔧 Диагностика`.

## 🔧 Диагностика

Единственное место для подробностей:

- `☁️ API` - лимиты, ошибки, последние ответы.
- `🤖 LLM` - провайдеры, fallback, токены.
- `🧠 Новости` - Tavily, фильтрация, кэш.
- `📜 Логи` - последние ошибки за 24 часа.

Скрытые команды быстрого доступа открывают те же экраны:

- `/admin_debug_api`
- `/admin_debug_llm`
- `/admin_debug_news`
- `/admin_logs`

## 🔔 Уведомления

Раздел показывает только рабочую сводку: статус, отправки за сегодня, ошибки за
сегодня, число активных типов уведомлений и время обновления.

Расписание, следующее уведомление, длинный список типов и история успешных
отправок здесь не показываются. Для ручной проверки есть переход в `🧪 Тесты`.

## 🧪 Тесты

Тестовые уведомления отправляются только администратору. Кнопки короткие и
разложены в два столбца:

- `☀️ Утро`
- `🌦 Погода`
- `📚 NL`
- `🇬🇧 EN`
- `🍽 Еда`
- `🎬 Досуг`
- `🧠 News`
- `✅ Все`

После запуска показывается короткий результат с временем, названием теста,
статусом и получателем.

## 👥 Пользователи

Экран показывает только сводку без технических идентификаторов:

- всего пользователей;
- активных за 7 дней;
- новых за 7 дней;
- пользователей с уведомлениями;
- админов;
- активные инвайты.

Кнопка `➕ Инвайт` стоит первой. Списки `user_id`, `chat_id`, последние сообщения
и технические поля не выводятся.

## 📜 Логи

Показывает только ошибки за последние 24 часа. Если ошибок нет, экран состоит из
короткого зелёного статуса и времени обновления. Обычные OK-события не выводятся.

## Callback data

Новая админская навигация использует короткие callback-и:

- `adm_home`
- `adm_check_all`
- `adm_system`
- `adm_system_check`
- `adm_diag`
- `adm_diag_api`
- `adm_diag_llm`
- `adm_diag_news`
- `adm_logs`
- `adm_notif`
- `adm_notif_check`
- `adm_users`
- `adm_invite`
- `adm_invite_create`
- `adm_welcome`
- `adm_welcome_edit`
- `adm_welcome_preview`
- `adm_tests`
- `adm_test_morning`
- `adm_test_weather`
- `adm_test_word_nl`
- `adm_test_word_en`
- `adm_test_recipe`
- `adm_test_leisure`
- `adm_test_news`
- `adm_test_all`
- `adm_test_repeat`

Старые `set_admin_*` callback-и оставлены как совместимость для уже отправленных
сообщений в Telegram. Они переводят пользователя на актуальные экраны или общий
fallback, не обходя `settings._admin_guard`.
