# Админ

Роль: администратор бота, владелец `config.CHAT_ID`.

Панель отвечает на четыре вопроса: всё ли работает, есть ли проблемы, кто
пользуется ботом и как быстро проверить уведомления вручную.

## Навигация

```text
🛠 Админ
├── 📊 Система
│   └── 📜 Логи
├── 🔔 Уведомления
└── 👥 Пользователи
    ├── ➕ Инвайт
    └── ✉️ Приветствие
```

Основных вкладок больше нет:

- `🔧 Диагностика`
- `🧪 Тесты`
- `☁️ API`
- `🤖 LLM`
- `🧠 Новости`
- `⚠️ Проблемы`

Старые callback-и этих вкладок оставлены как redirects, чтобы уже отправленные
сообщения Telegram не ломались.

## 🛠 Админ

Главный экран - короткая сводка без debug-шума:

- `📊 Система`
- `🔔 Уведомления`
- `👥 Пользователи`
- `💾 Данные`

На главном экране нет кнопки `🔄 Проверить всё`, отдельной кнопки `🧪 Тесты`,
расписания уведомлений, OpenWeather-лимитов, LLM-счётчиков, Tavily, Personal
News, последних API-вызовов, последних сбоев и OK-истории.

## 📊 Система

Единый экран для API, LLM, новостей и общей инфраструктуры. Одна строка - один
сервис или модуль, каждая строка начинается со статуса `🟢`, `🟡` или `🔴`.

Показываются:

- LLM
- OpenWeather
- Telegram
- TMDB
- Pexels
- Cloudflare
- Groq
- Gemini
- Tavily
- Новости
- Фото рецептов
- Данные
- Планировщик

OpenWeather отображается только в формате `6/500 сегодня`; слово `осталось` не
используется. Debug-метрики, кэш, fallback-логи, последние API-вызовы и
подробные списки ошибок в обычной навигации не показываются.

`🔄 Проверить систему` запускает существующие probe-проверки и остаётся на экране
`📊 Система`. `📜 Логи` открываются только отсюда.

Скрытые команды для подробной диагностики можно использовать напрямую:

- `/admin_debug_api`
- `/admin_debug_llm`
- `/admin_debug_news`
- `/admin_logs`

## 🔔 Уведомления

Экран объединяет состояние уведомлений и ручные тесты. Он показывает:

- статус;
- сколько отправлено сегодня;
- ошибки сегодня;
- активные типы;
- короткую сетку тестовых кнопок.

Тесты отправляются только администратору. Кнопки отсортированы по времени
плановой отправки, время стоит в начале названия, сетка идёт по 3 кнопки в ряд.
Отдельной вкладки `🧪 Тесты` больше нет.

## 👥 Пользователи

Экран показывает только сводку без `user_id`, `chat_id`, последних сообщений и
последней активности. Первая кнопка - `➕ Инвайт`.

`➕ Инвайт` сначала открывает экран описания, затем кнопка `✅ Создать` генерирует
одноразовую ссылку. `✉️ Приветствие` показывает текст, который увидит новый
пользователь после входа.

## 📜 Логи

Логи доступны из `📊 Система`. Экран показывает только ошибки за 24 часа. Если
ошибок нет, OK-события не выводятся.

## Callback data

Обычная навигация использует:

- `adm_home`
- `adm_system`
- `adm_system_check`
- `adm_logs`
- `adm_notif`
- `adm_notif_check`
- `adm_users`
- `adm_invite`
- `adm_invite_create`
- `adm_welcome`
- `adm_welcome_edit`
- `adm_welcome_preview`
- `adm_test_morning`
- `adm_test_weather`
- `adm_test_word_nl`
- `adm_test_word_en`
- `adm_test_recipe`
- `adm_test_leisure`
- `adm_test_news`
- `adm_test_all`
- `adm_test_repeat`

Redirects:

- `adm_check_all` -> `adm_system_check`
- `adm_diag` -> `adm_system`
- `adm_diag_api` -> `adm_system`
- `adm_diag_llm` -> `adm_system`
- `adm_diag_news` -> `adm_system`
- `adm_tests` -> `adm_notif`
- старые `set_admin_*` API/LLM/News callbacks -> `adm_system`

Во всех админских экранах кнопка назад называется одинаково: `⬅️ Назад`.
