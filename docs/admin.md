# Админ

Экран ошибок хранит записи максимум 12 часов. Одинаковые ошибки группируются с точным сообщением и количеством повторов; слишком большой список компактно обрезается, чтобы экран всегда открывался.

Роль: администратор бота, владелец `config.CHAT_ID`.

Панель отвечает на три вопроса: всё ли работает, есть ли проблемы и кто
пользуется ботом.

После запуска новой версии администратор получает одно уведомление с номером
версии и списком изменений. Строки состояния из `RELEASE_NOTES.md` (`Готово к
развёртыванию` и `Бот развёрнут и работает`) в список изменений не включаются:
итоговый статус `Бот развёрнут и работает ✅` показывается один раз внизу карточки.
В заголовке жирным выделяется только номер версии. После заголовка всегда идёт
одна пустая строка, затем подпись `Что изменено:` и 1–4 отдельных коротких
пункта: один пункт описывает одно изменение, без точки в конце. Итоговый статус
показывается обычным текстом.

## Навигация

```text
🛠 Админ
├── 🛠 Система
│   └── ⚠️ Ошибки
└── 👥 Пользователи
    ├── ➕ Инвайт
    └── ✉️ Приветствие
```

Основных вкладок больше нет:

- `🔧 Диагностика`
- `🧪 Тесты`
- `☁️ API`
- `🤖 LLM`
- `🧠 Новости`
- `⚠️ Проблемы`

Старые callback-и этих вкладок удалены. Кнопки из давно отправленных сообщений
не поддерживаются; актуальная навигация всегда открывается из текущей админки.

Экран `⚠️ Ошибки` открывается отдельной кнопкой из `🛠 Система`; скрытая команда
`/admin_logs` сохранена для быстрого доступа.

## 🛠 Админ

Главный экран показывает только один общий статус и время локального снимка:
`🟢 Всё работает · обновлено в 12:32`. При ограничениях строка становится жёлтой,
при пользовательской недоступности — красной. Если снимок старше 15 минут,
показывается `⚪ Состояние неизвестно · данные от 12:32`.

Состояния внешних сервисов читаются из сохранённого снимка фонового мониторинга.
При открытии выполняется только быстрый локальный запрос к PostgreSQL; провайдеры
последовательно не опрашиваются. Если последний снимок старше 15 минут, время
обновления получает пометку `данные устарели`.

На главном экране нет отдельных строк PostgreSQL, уведомлений, пользователей,
ошибок, квот и провайдеров. Кнопки `🛠 Система` и `👨🏻‍💻 Пользователи` стоят
в одном ряду; подробности открываются только по нажатию.

Внизу — `⬅️ Назад` (в личные настройки) и `#️⃣ Главная` (в главное меню бота), как
на всех экранах бота.

## 🛠 Система

Отдельный монитор проверяет AI-провайдеры, внешние API, Telegram и хранилище,
а экран только читает сохранённые состояния. Фоновый цикл запускается каждые
5 минут; результат любого обычного API-запроса обновляет то же состояние сразу.
Реальный запрос и фоновая проверка используют один переход состояния: квота,
ошибка, восстановление и выбранный резерв не могут расходиться между двумя
журналами.
Проверки, которые сами расходуют платную квоту (Spoonacular и ZeroEntropy),
выполняются не чаще раза в час; между ними монитор использует результаты
реальных обращений, чтобы диагностика не исчерпывала лимит сервиса.
Доступ к Google Books проверяется не чаще раза в сутки официальным поисковым
запросом одной книги; реальные обращения раздела обновляют тот же статус сразу.
При временной ошибке Google Books повторяет запрос до двух раз с короткой паузой,
после чего карточка книги продолжает работать через Open Library.
Если эта проверка завершилась временной сетевой или серверной ошибкой,
существующий монитор повторяет её через 5 минут. Ошибки ключа и доступа так не
повторяются.
Для Google Books дополнительно хранятся только текущая дата и число фактических
запросов за день; ответы из суточного кэша счётчик не увеличивают.
Для Cohere хранятся только текущий месяц и число фактических запросов Trial за
месяц; ответы из AI-кэша не учитываются. После 1 000 запросов Cohere не вызывается,
а структурные задачи сразу переходят в GitHub Models. Для коротких учебных задач
внутреннее рассуждение Cohere ограничено, чтобы лимит ответа оставался для результата.
Kimi удалён из активных провайдеров. Для общего резерва используются GitHub Models,
Groq, Cloudflare и OpenRouter в зависимости от раздела. Пустой или временно
неудачный ответ бесплатного OpenRouter повторяется один раз.
Каждый сервис хранит одинаковые поля: статус, подтверждённую квоту, выбранный
резерв, последнюю проверку, последний успех и последнюю ошибку.

Каждая строка имеет формат:

```text
эмодзи Сервис · раздел интерфейса · статус или статистика
```

Зелёная строка не повторяет слова «работает», «доступен» или «в норме».
Подтверждённая квота выводится как `N из M`, исчерпанная — как
`лимит исчерпан`. При значении не более 20% строка становится жёлтой. Если API
не сообщает фиксированную квоту, экран показывает фактический расход за текущий
период, ограничение скорости или технический предел. Неподтверждённые тарифы и
демонстрационные числа не подставляются. Для одного раздела показывается его
название, для двух и более — `Везде`.

Примеры полезных строк: Gemini показывает число запросов сегодня и модель,
LanguageTool — число проверок сегодня, Azure Speech — число символов сегодня,
Telegram — предел `до 30 сообщений/с`, а TheMealDB и REST Countries —
`без дневной квоты`. Groq, GitHub Models, OpenRouter, Spoonacular и другие API
используют квоту из ответа или rate-limit-заголовков, когда сервис её сообщает.
Ответ Spoonacular `402` показывается как исчерпанный дневной лимит, а не как
неопределённый статус.

Резерв проверяется до переключения и появляется в конце строки только после
реального выбора, без слов «используется резерв». Направленный граф резервов
проверяется на циклы; у поиска разрешено только направление Firecrawl → Tavily,
а у туристических фотографий — Pexels → Unsplash.

`⚪` означает, что сервис ещё не проверялся. `🟡` означает ограничение,
временную ошибку или работу через резерв.
`🔴` допустим только когда функция действительно не работает и рабочего
резерва нет. HTTP-коды, traceback, `NameError`, timeout и ошибки разбора JSON
на этом экране не показываются: они переводятся в понятные формулировки.
Для Google Books отдельно различаются неверный ключ, отключённый API, запрет
доступа, дневной лимит, слишком частые запросы, недоступность Google и отсутствие
соединения. При успешной проверке строка показывает локальный остаток:
`🟢 Google Books · Книги · 963 из 1 000`. При остатке не более 200 запросов
строка становится жёлтой. После 1 000 запросов Google больше не вызывается,
а строка сообщает, что используется Open Library.
Строка Cohere показывает месячный остаток без служебных слов:
`🟢 Cohere · Везде · 940 из 1 000`. При остатке не более 200 запросов она
становится жёлтой, а при исчерпанном лимите сообщает о GitHub Models.

Экран не показывает служебные строки про автоматический резерв, модель AI,
процентиль времени ответа и число медленных действий. Для каждого подключения
остаётся только актуальный статус, область использования и понятный остаток
лимита либо причина ограничения. Сырая последняя ошибка на экран не выводится.
Отдельная кнопка `⚠️ Ошибки` открывает технические детали; ниже остаётся
стандартная строка `⬅️ Назад | #️⃣ Главная`.

## 👥 Пользователи

Экран показывает сводку (без `user_id`, `chat_id`, последних сообщений), а ниже -
список пользователей: индикатор свежести (🟢/🟡/🔴), имя и когда последний раз
заходил. Список отсортирован по свежести активности, показывает до 15 записей;
если пользователей больше, внизу видно, сколько ещё не показано. Первая кнопка -
`➕ Инвайт`. Отдельной кнопки "Обновить" нет - открытие экрана уже показывает
свежие данные.

`➕ Инвайт` сначала открывает экран описания, затем кнопка `✅ Создать` генерирует
одноразовую ссылку. `✉️ Приветствие` показывает текст, который увидит новый
пользователь после входа.

## ⚠️ Ошибки

Экран показывает все реальные ошибки за 24 часа. Обычные успешные проверки,
переключение на резерв и восстановление не входят в счётчик ошибок и не создают
отдельные строки. События мониторинга имеют тип `error`, `fallback`, `recovery`,
`status` или `system` и связываются одним `incident_id`.

Цепочка «ошибка → резерв → восстановление» показывается одной записью. В ней
сохраняются первичная причина, HTTP-код или тип исключения, короткое сообщение,
длительность запроса, время начала, выбранный резерв и время восстановления.
Лимит основного AI-провайдера не помечается как потерянный ответ, если запрос
успешно завершил резерв.
В начале экрана «Система» показывается строка `Ответы`: время, в которое
укладываются 95% действий за последние сутки, и число действий дольше своего
лимита. Содержание сообщений и ответов в этой статистике не хранится.
Лимит сервиса считается восстановленным только после реального успешного запроса:
служебная проверка доступности не закрывает квотный инцидент. Для Tavily ответ
`HTTP 432` показывается как исчерпанный лимит тарифа.
Повторная успешная проверка при неизменном состоянии в историю не записывается.
Ошибка приложения занимает одну строку: время, раздел, действие пользователя,
текст ошибки и место в коде. Отдельных кнопок копирования нет.
Время каждой записи показывается в часовом поясе бота `Europe/Amsterdam`, а не
в системном UTC-времени сервера Railway.
Одинаковые ошибки фоновой проверки одного сервиса, возникшие в пределах десяти
минут, показываются одной строкой со счётчиком повторов. Ticketmaster проверяется
в фоне не чаще одного раза в час, поэтому открывать раздел «Концерты» для такой
проверки не требуется.

Секреты перед сохранением и повторно перед показом заменяются на `[REDACTED]`.
Открытие экрана сохраняет для каждого администратора позицию просмотра; главный
экран после этого показывает только появившиеся позже ошибки. Кнопка `❌ Очистить ошибки`
очищает ошибки приложений и инциденты сервисов. Навигация остаётся
стандартной: `⬅️ Назад | #️⃣ Главная`.
Для временных сетевых сбоев Telegram, AI и внешних API в логах сохраняются
короткие технические детали операции и попытки восстановления; сырые внутренние
имена файлов остаются внутри подробностей инцидента.

Например: `🟡 Gemini · Везде · лимит исчерпан · GitHub Models`. Raw-ответы API,
stack trace и тело ошибки на главном экране не показываются.

Обычная навигация использует:

- `adm_home`
- `adm_api_ai`
- `adm_system`
- `adm_logs`
- `adm_logs_clear`
- `adm_users`
- `adm_user_del`
- `adm_user_delconfirm_*`
- `adm_user_delok_*`
- `adm_invite`
- `adm_invite_create`

Callbacks удалённых экранов и кнопок из старых сообщений больше не
перенаправляются: контрактом считается только текущая админская навигация.

Во всех админских экранах кнопка назад называется одинаково: `⬅️ Назад`.
