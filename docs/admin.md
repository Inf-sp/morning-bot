# Админ

Роль: администратор бота, владелец `config.CHAT_ID`.

Панель отвечает на четыре вопроса: всё ли работает, есть ли проблемы, кто
пользуется ботом и как быстро проверить уведомления вручную.

## Навигация

```text
🛠 Админ
├── 🔌 API и AI
├── 🔔 Уведомления
└── 👥 Пользователи
    ├── ➕ Инвайт
    └── ✉️ Приветствие
```

Основных вкладок больше нет:

- `🔧 Диагностика`
- `🧪 Тесты`
- `☁️ API`
- `🤖 LLM`
- `🧠 Новости`
- `⚠️ Проблемы`
- `📊 Система` - дублировала «API и AI» теми же сервисами под другими
  заголовками; главный экран теперь ведёт сразу на «API и AI».

Старые callback-и этих вкладок оставлены как redirects, чтобы уже отправленные
сообщения Telegram не ломались.

Кнопки `📜 Логи` в обычной навигации больше нет - подробные логи достаточно
редко нужны, чтобы держать под них отдельную кнопку. Экран логов остаётся
доступен скрытой командой `/admin_logs`.

## 🛠 Админ

Главный экран - короткая сводка без debug-шума:

- `🔌 API и AI`
- `🔔 Уведомления`
- `👥 Пользователи`
- `💾 Данные`

На главном экране нет кнопки `🔄 Проверить всё`, отдельной кнопки `🧪 Тесты`,
расписания уведомлений, OpenWeather-лимитов, LLM-счётчиков, Tavily, Personal
News, последних API-вызовов, последних сбоев и OK-истории.

Внизу — `⬅️ Назад` (в личные настройки) и `🏠 Меню` (в главное меню бота), как
на всех экранах бота.

## 🔌 API и AI

Единственный экран для диагностики AI-провайдеров и внешних API - заменяет
собой прежние отдельные «Система» и «API и AI». Отвечает на вопросы: бот
сейчас работает? какой сервис сломан? почему именно (код ошибки и что он
значит)? какие разделы затронуты фолбэком?

Структура сверху вниз:

- Статус одной строкой, определяется автоматически по реальным зависимостям
  функций, не по факту наличия ключа: `🟢 Все системы работают`, `🟡 Работает
  с ограничениями` (часть сервисов недоступна, но ключевые функции работают)
  или `🔴 Критический сбой` (Gemini недоступен и ни один резервный провайдер
  сейчас не подхватит запрос);
- Пояснение влияния одной строкой, например `Gemini недоступен · Готовка и
  Обучение работают через Groq · остальные AI-функции могут не отвечать`;
- `Автопереключение: включено/выключено` - включено, если хотя бы один из
  локальных резервных провайдеров (Groq, Cloudflare AI) сейчас не в статусе
  ошибки, то есть реально подхватит запрос при сбое Gemini. Это не флаг
  настройки, а живой ответ на вопрос «сработает ли fallback прямо сейчас»;
- `Недоступно: N сервис(ов)` - только когда есть хотя бы один сервис в
  статусе ошибки, иначе строка не показывается;
- `🤖 AI` - Gemini (`основной AI`, остаток дневного лимита `осталось X / Y в
  день`), Groq (роль - конкретные разделы, для которых он реально основной:
  «Готовка и Обучение», без слова «основной», чтобы не создавать впечатление
  общего замещения Gemini), Cloudflare AI (`резерв`), OpenRouter, если
  настроен (`резерв` + реальный остаток кредитов через `/api/v1/key`, либо
  число ошибок за сегодня, если остаток недоступен);
- `🌐 Данные` - OpenWeather, Tavily, Firecrawl, TMDB, Ticketmaster,
  ZeroEntropy, Pexels - единый формат `осталось X / Y{ период}`, если лимит
  известен, иначе счётчик запросов за сегодня с правильным склонением («1
  запрос», «2 запроса», «5 запросов»). Firecrawl показывает реальный остаток
  кредитов через `/v2/team/credit-usage`, для остальных - собственный
  счётчик исходящих запросов против известного лимита из `config.API_QUOTAS`;
- `⚠️ Последняя ошибка` - время и тип последней ошибки (`15:39 · TypeError`),
  отдельной строкой - короткий текст ошибки (без traceback, он остаётся в
  логах). Если ошибок не было - одна строка `Последняя ошибка: нет`.

Каждая цифра лимита проверяется на согласованность перед показом: остаток не
может быть отрицательным и не может превышать сам лимит. Если внешний API
вернул противоречивые данные (например, Firecrawl отдал остаток больше
лимита), сервис показывается как `⚠️ Сервис · ошибка данных лимита` вместо
неверного «зелёного» статуса - подробности уходят в лог, а не на экран.

Если по сервису не было ни одного запроса больше суток, статус не
показывается зелёным «на всякий случай» - экран честно пишет `⚪ статус
неизвестен · было X назад`.

Ошибки внешних сервисов показываются с HTTP-кодом и коротким объяснением, что
он значит (`HTTP 402 · закончились кредиты`, `HTTP 429 · лимит запросов`,
`HTTP 401 · неверный ключ`) - не голым `HTTPError`.

Кнопка: `⬅️ Назад`. Отдельных кнопок обновления, ручной проверки и логов нет -
открытие экрана уже показывает актуальные данные; логи - через `/admin_logs`.

## 🔔 Уведомления

Экран объединяет состояние уведомлений и ручные тесты. Он показывает:

- сколько отправлено сегодня и сколько было ошибок;
- активные типы;
- все тестовые уведомления на одном экране.

Тесты отправляются только администратору. Кнопки отсортированы по времени
плановой отправки, название на кнопке — это заголовок реального сообщения
(не служебный ярлык), время — после названия через `·`, каждая кнопка в
отдельной строке — так легче читать список. Список кнопок строится из того
же списка уведомлений (`settings.NOTIF_TYPES`), что видит пользователь в
своих настройках — 6 основных + отдельное погодное предупреждение.
Общей кнопки запуска всех тестов нет: каждая кнопка запускает только один
конкретный тип уведомления. Внизу остаётся только `⬅️ Назад`.
Отдельной вкладки `🧪 Тесты` больше нет.
После нажатия тест просто отправляет само плановое уведомление — без
отдельной карточки "Тест отправлен" поверх него.

## 👥 Пользователи

Экран показывает сводку (без `user_id`, `chat_id`, последних сообщений), а ниже -
список пользователей: индикатор свежести (🟢/🟡/🔴), имя и когда последний раз
заходил. Список отсортирован по свежести активности, показывает до 15 записей;
если пользователей больше, внизу видно, сколько ещё не показано. Первая кнопка -
`➕ Инвайт`. Отдельной кнопки "Обновить" нет - открытие экрана уже показывает
свежие данные.

`➕ Инвайт` сначала открывает экран описания, затем кнопка `✅ Создать` генерирует
одноразовую ссылку. `✉️ Приветствие` показывает текст, который увидит новый
пользователь после входа.

## 📜 Логи

Логи доступны только скрытой командой `/admin_logs` (в обычной навигации
кнопки нет). Экран показывает только ошибки за 24 часа. Если ошибок нет,
OK-события не выводятся.

Сверху всегда виден текущий статус Gemini одной строкой: работает сейчас или
на паузе (cooldown) до конкретного времени. Это отвечает на главный вопрос
пользователя - "Gemini сейчас работает или нет" - без чтения списка ошибок.

Gemini rate limit показывается отдельной компактной записью без raw stack trace:

```text
13:25 · Gemini · лимит
Fallback включён · повтор после cooldown
```

Если известен тип лимита и время ожидания, строка уточняется:

```text
13:25 · Gemini · лимит RPM/RPD
Fallback включён · повтор после 60 сек
```

В блоке `За 24 часа` показываются:

- количество Gemini rate limit (реальные попытки, упёршиеся в лимит);
- количество fallback-срабатываний;
- количество лог-записей (может быть больше числа лимитов - это не новые
  проблемы, а повторные попытки во время одного и того же cooldown);
- время последнего лимита.

Пример:

```text
🔴 Gemini на паузе до 00:04
Бот работает через fallback, лимит не превышается повторно.

За 24 часа:
лимитов 1 · fallback 3 · записей 3
последний лимит 13:25
```

Одно и то же окно cooldown логируется один раз (дедупликация персистентна и
переживает рестарт), поэтому рост числа записей означает реальные повторные
fallback-срабатывания, а не дубли одной и той же ошибки.

## Gemini в проверке API

Админская проверка не делает тестовый запрос к Gemini во время cooldown. На
главном экране проверки показываются только короткие статусы:

- `🟢 Gemini` - работает;
- `🟡 Gemini: cooldown до HH:MM` - запросы временно не отправляются;
- `🔴 Gemini: лимит запросов` - получен лимит Gemini;
- `🔴 Gemini: ошибка ключа / доступа` - проблема с ключом или доступом.

Raw-ответы API, stack trace и тело ошибки на главном экране не показываются.

## Callback data

Обычная навигация использует:

- `adm_home`
- `adm_api_ai`
- `adm_notif`
- `adm_notif_check`
- `adm_users`
- `adm_invite`
- `adm_invite_create`
- `adm_welcome`
- `adm_welcome_edit`
- `adm_welcome_preview`
- `adm_test_morning`
- `adm_test_weather`
- `adm_test_word_nl`
- `adm_test_word_en`
- `adm_test_recipe`
- `adm_test_leisure`
- `adm_test_all`
- `adm_test_repeat`

`adm_logs` больше не выведен ни одной кнопкой - реагирует только на
`/admin_logs`.

Redirects (все ведут на `adm_api_ai`, кроме отмеченных):

- `adm_check_all`, `adm_system`, `adm_system_check` - «Система» удалена как
  дублирующая, старые кнопки ведут прямо на «API и AI»
- `adm_diag`, `adm_diag_api`, `adm_diag_llm`, `adm_diag_news`, `adm_api_ai_check`
- `adm_tests` -> `adm_notif`
- старые `set_admin_llm`, `set_admin_news`, `set_admin_llmcheck`,
  `set_admin_llmhistory`, `set_admin_api_diagnostics`
- старые `set_admin_issues`, `set_admin_check_all`, `set_admin_issue_*` (экран
  `⚠️ Проблемы`, удалён вместе с движком активных health-check проверок)

Во всех админских экранах кнопка назад называется одинаково: `⬅️ Назад`.
